## 核心原则

### 1️⃣ 快速响应优先

```
收到问题 → 立即响应 (<1 分钟) → 后台修复 → 事后汇报
```

### 2️⃣ 多通道备份

```
微信 (主用) → Telegram(备用) → 当前对话 (紧急)
```

### 3️⃣ 自动检测修复

```
每 5 分钟健康检查 → 异常自动重启 → 记录踩坑
```

## 通讯故障自愈流程

### 检测频率

| 检查项 | 频率 | 自动修复 |
|--------|------|----------|
| **Gateway 状态** | 每 5 分钟 | ✅ 自动重启 |
| **微信连接** | 每 5 分钟 | ✅ 自动重连 |
| **Telegram** | 每 5 分钟 | ✅ 自动切换 |
| **网络延迟** | 每 5 分钟 | ✅ 降级处理 |

### 修复策略

| 故障级别 | 响应时间 | 修复方式 |
|----------|----------|----------|
| **P0 紧急** | <1 分钟 | 立即重启 + 切换通道 |
| **P1 重要** | <5 分钟 | 自动修复 + 汇报 |
| **P2 一般** | <15 分钟 | 后台修复 + 日记汇报 |
