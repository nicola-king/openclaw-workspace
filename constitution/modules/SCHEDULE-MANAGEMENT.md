## 🔄 五大机制（彻底解决）

### 1️⃣ 任务注册机制

**所有定时任务必须预先注册：**

```bash
# 任务注册文件
~/.taiyi/schedule/registered-tasks.json
```

**注册内容：**
```json
{
  "task_id": "TASK-XXX",
  "name": "任务名称",
  "cron": "0 12 * * *",
  "bot": "太一",
  "priority": "P0",
  "deadline": "12:05",
  "report_to": "SAYELF",
  "status": "active"
}
```

**验证流程：**
- [ ] Cron 配置存在
- [ ] 脚本路径正确
- [ ] 日志目录存在
- [ ] 8 步流程模板就绪

---

### 2️⃣ 执行监控机制

**每分钟检查任务状态：**

```bash
# 监控脚本
~/.taiyi/schedule/monitor.sh
```

**监控内容：**
- [ ] 任务是否准时触发
- [ ] 任务是否执行完成
- [ ] 日志是否生成
- [ ] 上报是否发送

**状态追踪：**
```json
{
  "task_id": "TASK-XXX",
  "scheduled": "12:00:00",
  "started": "12:00:01",
  "completed": "12:04:30",
  "reported": "12:05:00",
  "status": "✅ success"
}
```

---

### 3️⃣ 失败告警机制

**任务失败立即告警：**

**告警级别：**
| 级别 | 触发条件 | 告警方式 |
|------|---------|---------|
| **P0** | 任务超时 5 分钟 | Telegram + 日志 |
| **P1** | 任务超时 10 分钟 | Telegram |
| **P2** | 任务超时 30 分钟 | 日志 |

**告警内容：**
```
⚠️ 定时任务失败告警

任务 ID: TASK-XXX
任务名称：12:00 午间汇报
计划时间：12:00:00
当前时间：12:05:00
超时：5 分钟
执行 Bot: 太一
状态：❌ 未执行

立即执行补做流程！
```

---

### 4️⃣ 自动补做机制

**任务失败自动补做：**

**补做流程：**
1. 检测到任务失败
2. 立即触发补做
3. 8 步流程上报
4. 记录失败原因
5. 优化防止再犯

**补做模板：**
```markdown
【补做任务 · YYYY-MM-DD HH:mm】

任务 ID: TASK-XXX
原计划：HH:mm
补做时间：HH:mm
延迟：X 分钟
原因：[根本原因]
整改：[预防措施]

8 步流程：
1. 任务分解 ✅
2. 分发员工 ✅
...
8. 归档分发 ✅
```

---

### 5️⃣ 主动上报机制

**不等待 SAYELF 提醒，主动上报：**

**上报时间：**
- 任务完成后 5 分钟内
- 不等待 SAYELF 检查
- 不等待 heartbeat 询问

**上报内容：**
```markdown
【定时任务完成汇报】

任务 ID: TASK-XXX
任务名称：[名称]
计划时间：HH:mm
完成时间：HH:mm
执行 Bot: [Bot]
状态：✅ 完成

8 步流程：✅ 全部完成
成果文件：[文件路径]
分发状态：✅ 已发送 SAYELF
```
