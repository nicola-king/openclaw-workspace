# 记忆链接模块

**层级：** Tier 3 (术层)
**生效：** 立即生效

---

## 核心原则

### 1️⃣ 不忘事

```
✅ 新会话扫描旧会话
✅ 关键词联系回滚
✅ 记忆连续性保持
✅ 待办事项追踪
```

### 2️⃣ 不幻觉

```
✅ 只链接已验证记忆
✅ 不编造不存在的内容
✅ 所有链接有来源
✅ 无法验证→标注存疑
```

---

## 新会话启动流程

```
00:00 新会话开始
    ↓
1. 扫描昨日 memory/YYYY-MM-DD.md
    ↓
2. 提取关键词/待办/任务
    ↓
3. 回滚到 HEARTBEAT.md
    ↓
4. 验证记忆链接
    ↓
5. 加载已验证记忆
    ↓
6. 开始新会话
```

---

## 关键词联系机制

### 关键词类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **任务关键词** | 进行中的任务 | 知几-E/跨境外贸/微信 |
| **项目关键词** | 持续的项目 | 小红书运营/ODA 注册 |
| **时间关键词** | 定时任务 | 23:00 日记/09:00 推送 |
| **决策关键词** | SAYELF 决策 | 模拟盘启动/微信接入 |

### 回滚规则

| 规则 | 说明 |
|------|------|
| **7 天内记忆** | 全部回滚 |
| **30 天内记忆** | 关键词匹配回滚 |
| **30 天 + 记忆** | 仅 HEARTBEAT.md |

---

## 记忆验证机制

### 验证标准

| 验证项 | 标准 |
|--------|------|
| **来源明确** | 有 memory 文件记录 |
| **时间明确** | 有日期/时间戳 |
| **SAYELF 确认** | 有 SAYELF 决策记录 |
| **可验证** | 有外部链接/截图 |

### 标注规则

| 状态 | 标注 | 说明 |
|------|------|------|
| **已验证** | ✅ | 可安全链接 |
| **待验证** | 🟡 | 谨慎链接 |
| **存疑** | ❌ | 不链接 |

---

## 记忆文件结构

```
memory/
├── YYYY-MM-DD.md (每日会话记录)
├── diary/YYYY-MM-DD.md (Agent 日记)
├── PITFALLS.md (踩坑日志)
├── HEARTBEAT.md (待办事项)
└── MEMORY.md (长期记忆)
```

---

## 不忘事机制

### 待办追踪

```
旧会话待办 → HEARTBEAT.md → 
新会话加载 → 继续执行
```

### 任务连续性

```
任务开始 → 记录 memory → 
会话终结 → 待办转移 → 
新会话 → 加载继续
```

---

## 不幻觉机制

### 禁止行为

```
❌ 编造不存在的记忆
❌ 夸大已验证的内容
❌ 脱离 memory 文件
❌ 自我推断无来源
```

### 验证流程

```
记忆声明 → 检查 memory 文件 → 
有来源 → ✅ 可链接
无来源 → ❌ 标注存疑
```

---

## 示例

### ✅ 正确链接

```
新会话：
"知几-E 模拟盘已启动 (2026-03-23 10:03)"

验证：
memory/2026-03-23.md 有记录 ✅
→ 可安全链接
```

### ❌ 错误链接

```
新会话：
"知几-E 已盈利$1000"

验证：
memory 文件无记录 ❌
→ 标注存疑，不链接
```
