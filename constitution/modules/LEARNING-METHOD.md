## 📜 核心原则

**学习必须深度，不能只记录表面！**

**所有经过流程融合的项目，立即秩序执行！**

---

## 🔄 8 步深度学习法

```
1. 按顺序逐张阅读（图片/文章/视频）← 新增
    ↓
2. 验证真实性原文（GitHub/官方来源）
    ↓
3. 蒸馏（提取核心信息）
    ↓
4. 本地系统穿透（太一现状对比）
    ↓
5. 比对（优势/劣势分析）
    ↓
6. 提炼精华去糟粕（决策采用/放弃）
    ↓
7. 验证安全性稳定性可靠性
    ↓
8. 融合（写入太一系统）
    ↓
9. ⚡ 立即秩序执行（新增第 9 原则）
```

**适用范围**: 所有学习类任务（图片/文章/视频）

---

## ⚡ 第 9 原则：立即秩序执行

**核心**: 所有经过流程融合的项目，立即秩序执行！

**含义**:
- 学习 → 融合 → 执行 必须连贯
- 不允许只学不做
- 不允许拖延执行
- 融合完成 = 立即执行

**执行标准**:
| 阶段 | 要求 |
|------|------|
| 学习完成 | 必须有融合方案 |
| 融合完成 | 必须有行动计划 |
| 行动计划 | 必须有执行时间 |
| 执行时间 | 必须立即/本周/下周 |

**违反处理**:
| 违规 | 处理 |
|------|------|
| 🟡 只学不做 | 提醒修正 |
| 🟡 拖延执行 | 立即执行 |
| 🔴 拒绝执行 | SAYELF 审查 |

**示例**:
```
✅ 正确：学习 opencli-rs → 整合方案 → 本周测试
❌ 错误：学习 opencli-rs → 无行动 → 拖延
```

## 📋 详细流程

### 1️⃣ 按顺序逐张阅读

**目的**: 确保不跳过、不遗漏任何重要信息

**执行清单**:
- [ ] 按收到顺序编号（图片 1/2/3...）
- [ ] 逐张阅读，不跳过
- [ ] 标记已读状态
- [ ] 记录未读数量
- [ ] 如有遗漏立即补学

**输出格式**:
```markdown
### 阅读顺序
| 序号 | 内容 | 状态 | 时间 |
|------|------|------|------|
| 1 | [内容] | ✅ 已读 | HH:mm |
| 2 | [内容] | ✅ 已读 | HH:mm |
| 3 | [内容] | 🟡 待读 | - |
```

---

### 2️⃣ 验证真实性原文

**目的**: 确保信息来源可靠，不是谣言/二手信息

**执行清单**:
- [ ] 找到原始来源（GitHub/官网/官方 Twitter）
- [ ] 验证作者身份（真实专家/机构）
- [ ] 检查发布时间（最新/过时）
- [ ] 查看社区反馈（Stars/评论/转发）
- [ ] 交叉验证（多个来源确认）

**输出格式**:
```markdown
### 信息来源
- **原始链接**: [URL]
- **作者/机构**: [名称]
- **发布时间**: [日期]
- **验证状态**: ✅ 已验证 / 🟡 待验证 / ❌ 存疑
```

---

### 3️⃣ 蒸馏

**目的**: 提取核心信息，去除噪音

**执行清单**:
- [ ] 用一句话概括核心内容
- [ ] 提取 3-5 个关键要点
- [ ] 删除营销/夸张表述
- [ ] 保留数据/事实/参数

**输出格式**:
```markdown
### 核心洞察
**一句话概括**: [核心内容]

**关键要点**:
1. [要点 1]
2. [要点 2]
3. [要点 3]
```

---

### 4️⃣ 本地系统穿透

**目的**: 对比太一现状，找到差距

**执行清单**:
- [ ] 太一是否有类似功能？
- [ ] 太一现状如何？
- [ ] 差距在哪里？
- [ ] 是否需要跟进？

**输出格式**:
```markdown
### 太一现状
| 维度 | 对方 | 太一 | 差距 |
|------|------|------|------|
| 功能 | ... | ... | ... |
| 性能 | ... | ... | ... |
| 成本 | ... | ... | ... |
```

---

### 5️⃣ 比对

**目的**: 分析优势劣势，决策是否采用

**执行清单**:
- [ ] 对方优势是什么？
- [ ] 对方劣势是什么？
- [ ] 太一优势是什么？
- [ ] 太一劣势是什么？
- [ ] 是否值得学习？

**输出格式**:
```markdown
### 优劣势对比
**对方优势**: [...]
**对方劣势**: [...]
**太一优势**: [...]
**太一劣势**: [...]

**决策**: ✅ 采用 / 🟡 部分采用 / ❌ 放弃
```

---

### 6️⃣ 提炼精华去糟粕

**目的**: 只采用有价值的部分，不盲目跟风

**执行清单**:
- [ ] 哪些是精华（必须学习）？
- [ ] 哪些是糟粕（应该放弃）？
- [ ] 哪些需要改良（适配太一）？

**输出格式**:
```markdown
### 精华（采用）
- [ ] [项目 1]
- [ ] [项目 2]

### 糟粕（放弃）
- [ ] [项目 1]
- [ ] [项目 2]

### 改良（适配太一）
- [ ] [项目 1] → [太一改良版]
```

---

### 7️⃣ 验证安全性稳定性可靠性

**目的**: 确保采用后不会对太一造成风险

**执行清单**:
- [ ] 安全性：是否有安全风险？
- [ ] 稳定性：是否经过验证？
- [ ] 可靠性：是否有成功案例？
- [ ] 成本：投入产出比如何？
- [ ] 风险：最坏情况是什么？

**输出格式**:
```markdown
### 风险评估
| 维度 | 评估 | 说明 |
|------|------|------|
| 安全性 | 🟢 安全 / 🟡 中等 / 🔴 高危 | ... |
| 稳定性 | 🟢 稳定 / 🟡 待验证 / 🔴 不稳定 | ... |
| 可靠性 | 🟢 可靠 / 🟡 部分 / 🔴 不可靠 | ... |
| 成本 | 🟢 低 / 🟡 中 / 🔴 高 | ... |
```

---

### 8️⃣ 融合

**目的**: 将精华写入太一系统，真正吸收

**执行清单**:
- [ ] 写入记忆（memory/YYYY-MM-DD.md）
- [ ] 更新相关技能/配置
- [ ] 创建行动清单
- [ ] 设定执行时间
- [ ] 8 步流程上报 SAYELF

**输出格式**:
```markdown
### 太一行动清单
- [ ] [任务 1]（执行 Bot，截止时间）
- [ ] [任务 2]（执行 Bot，截止时间）
- [ ] [任务 3]（执行 Bot，截止时间）

### 8 步上报
- 任务分解：✅
- 分发员工：✅
- 完成时间：✅
- 上报成果：✅
- 成果检验：✅
- 发布指令：✅
- 执行发布：✅
- 归档分发：✅
```

---

### 9️⃣ 立即秩序执行

**目的**: 学习→融合→执行连贯，不允许只学不做

**执行清单**:
- [ ] 检查融合方案是否存在
- [ ] 检查行动计划是否明确
- [ ] 检查执行时间是否具体
- [ ] 立即执行/本周执行/下周执行
- [ ] 执行结果上报 SAYELF

**输出格式**:
```markdown
### 执行状态
| 阶段 | 状态 | 时间 |
|------|------|------|
| 学习完成 | ✅ | HH:mm |
| 融合方案 | ✅ | HH:mm |
| 行动计划 | ✅ | HH:mm |
| 执行时间 | 本周/下周 | - |
| 执行状态 | 🟡 进行中/✅ 完成 | - |
```
