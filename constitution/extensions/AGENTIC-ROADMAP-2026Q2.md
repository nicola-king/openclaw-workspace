## 🛠️ 技术架构升级

### 当前架构 (2026.Q1)

```
太一 (总管)
├── 知几 (交易)
├── 山木 (内容)
├── 素问 (技术)
├── 罔两 (数据)
└── 庖丁 (预算)

通信：Telegram Bot API
存储：本地文件 + Syncthing
执行：Python 脚本
```

### 升级架构 (2026.Q2)

```
太一 Agentic 系统
├── 感知层 (Perception)
│   ├── Telegram 消息监听
│   ├── Polymarket 数据流
│   ├── 链上数据监控 (Solana)
│   └── Web 搜索集成
├── 决策层 (Decision)
│   ├── 协议引擎 (宪法规则)
│   ├── 风险评估 (授权机制)
│   └── 任务调度 (委派协议)
├── 执行层 (Action)
│   ├── 交易执行 (知几-E)
│   ├── 内容发布 (山木)
│   ├── 文件同步 (Syncthing)
│   └── 邮件发送 (Mutt)
└── 学习层 (Learning)
    ├── 记忆固化 (TurboQuant)
    ├── 协议进化 (每周回顾)
    └── 技能涌现 (组合创新)

通信升级:
- 增加 Solana RPC 连接
- 增加 Gumroad API
- 增加 GitHub API
```
