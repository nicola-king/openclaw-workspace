## 🔧 委派流程

### 模式 1: 太一 → 专业 Bot

```
SAYELF → 太一：目标和方向
太一 → 专业 Bot：任务和方法
专业 Bot → 太一：结果
太一 → SAYELF：决策建议
```

**示例**:
```
SAYELF: "做 Polymarket 套利"

太一 → 知几:
  目标：发现套利机会
  约束：置信度>96%，优势>2%
  验收：3 个可行机会

知几 → 太一:
  机会 1: BTC 跨市场套利 (6.4%)
  机会 2: ETH 跨时间套利 (3.2%)
  机会 3: 结算收割 (2.8%)

太一 → SAYELF:
  建议：执行机会 1 (风险最低)
  需要授权：P1 级别 (¥50 预算)
```

---

### 模式 2: TurboQuant 智能分离

```
太一 (主成分) → 目标/约束/验收标准
    ↓
专业 Bot (残差) → 实现/细节/边界
    ↓
太一整合 → 完整方案
```

**示例**:
```
太一 → 山木 (主成分):
  主题：PolyAlert 介绍文章
  风格：极简黑客风
  长度：1800 字
  截止：今天 20:00

山木 → 太一 (残差):
  标题：《PolyAlert: 7 天上线的预测市场警报服务》
  大纲：问题→方案→实现→价值
  配图：3 张 (架构/流程/价值)
  关键词：Polymarket, Alert, Automation

太一整合:
  ✅ 符合主题和风格
  ✅ 长度达标
  ✅ 技术细节准确
  → 批准发布
```

## 📊 任务优先级

| 级别 | 名称 | 授权 | 响应 |
|------|------|------|------|
| **P0** | 紧急 | 事后报备 | 立即 |
