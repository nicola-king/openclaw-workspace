# 任务委派协议

## 委派命令格式

```
/委派 [Bot 名] [任务描述]
```

## 委派流程

1. **太一接收** → 解析任务，判断职责域
2. **切换频道** → `[频道：XXX]` 标注
3. **执行任务** → 工具 Bot 在职责域内处理
4. **结果汇报** → 工具 Bot → 太一
5. **整合输出** → 太一 → Nicola

## 委派确认格式

```
【委派确认】
任务：[任务描述]
执行频道：[Bot 名]
预计耗时：[时间]
交付物：[格式]

开始执行 → 回复「收到」
取消委派 → 回复「取消」
```

## 各 Bot 委派入口

| 命令 | 频道 | 职责 |
|------|------|------|
| /委派知几 | 知几 | 数据查询、分析、趋势 |
| /委派素问 | 素问 | 知识研究、调研、原理 |
| /委派庖丁 | 庖丁 | 财务拆解、成本、利润 |
| /委派山木 | 山木 | 项目执行、跟进、落地 |
| /委派罔两 | 罔两 | 市场监控、竞品、情报 |

## 跨频道协作

当任务涉及多个频道：
1. 太一拆解为子任务
2. 分别委派给对应频道
3. 汇总各频道结果
4. 整合后交付 Nicola

## 委派撤销

Nicola 可随时回复「停止」中断委派任务。
