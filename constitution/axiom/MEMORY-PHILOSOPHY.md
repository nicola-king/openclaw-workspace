# 记忆哲学（Memory Philosophy）

不是记住更多，而是记住正确的事、在正确层级、以正确成本。

---

## 核心公式

```
记忆效率 = (检索精度 × 访问速度) / 存储成本
记忆价值 = 未来访问概率 × 决策影响力
```

**目标：** 最大化记忆价值，最小化存储成本。

---

## 四层记忆架构（参考 Hermes）

| 层级 | 名称 | 内容 | 访问成本 | 更新频率 |
|------|------|------|---------|---------|
| **层 1** | 热记忆（提示内） | 身份锚点、核心原则 | 零成本 | 每次 Session |
| **层 2** | 温记忆（索引层） | 元数据、关键词、摘要 | 低成本 | 每日更新 |
| **层 3** | 冷记忆（文件层） | 完整对话记录、日志 | 中成本 | 按需读取 |
| **层 4** | 归档记忆（长期） |  curated 洞察、决策 | 一次性 | 每周提炼 |

---

## 太一系统的四层映射

### 层 1：热记忆（Hot Memory）

**文件：** `SOUL.md` + `constitution/axiom/*.md`

**内容：**
- 身份锚点（我是太一）
- 价值基石（创造价值才输出）
- 负熵法则（禁止冗余）
- 观察者协议（Nicola 是决策者）

**访问模式：** 每次 Session 无条件加载

---

### 层 2：温记忆（Warm Memory）

**文件：** `memory/index/registry.json`

**内容：**
- 记忆条目元数据（ID、时间、类型、关键词）
- 快速检索索引
- 重要性标记（P0/P1/P2）

**访问模式：** 需要检索时先查索引

**示例条目：**
```json
{
  "id": "TASK-20260323-001",
  "timestamp": 1711180800,
  "type": "decision",
  "keywords": ["微信接入", "openclaw-weixin"],
  "summary": "微信插件成功接入，私聊正常，群聊不支持",
  "file": "memory/2026-03-23.md",
  "importance": "P0"
}
```

---

### 层 3：冷记忆（Cold Memory）

**文件：** `memory/YYYY-MM-DD.md`

**内容：**
- 每日原始对话记录
- 任务执行日志
- 临时 TODO 和待办

**访问模式：** 按需读取完整文件

**保留策略：** 保留 30 天，之后归档或删除

---

### 层 4：归档记忆（Archived Memory）

**文件：** `MEMORY.md`

**内容：**
- 长期价值洞察
- 重大决策记录
- 用户偏好和习惯
- 系统演进历史

**访问模式：** 仅在主 Session 加载

**更新机制：** 每周提炼一次

## 记忆刷新协议

### 触发条件

| 条件 | 动作 |
|------|------|
| Session 结束 | 压缩关键内容到当日 memory |
| 每日 23:00 | 扫描当日 memory，提炼到 MEMORY.md |
| 每周日 23:00 | 汇总本周 memory，生成周报 |
| 索引超过 100 条 | 自动聚类，生成主题索引 |

## 遗忘机制

**原则：** 不遗忘 = 负熵违规（积累混乱）

### 自动遗忘

| 类型 | 保留期 | 动作 |
|------|--------|------|
| 临时 TODO | 7 天 | 未完成→删除或转移 |
| 对话记录 | 30 天 | 归档或删除 |
| 索引条目 | 90 天 | 无访问→删除 |

### 手动遗忘

SAYELF 可随时命令：
- `/memory clean` — 清理过期记忆
- `/memory archive` — 归档旧记录
- `/memory purge [关键词]` — 删除特定主题

---

## 检索策略

### 三级检索

```
1. 先查索引层 (registry.json) → 找到候选文件
2. 再读文件层 (YYYY-MM-DD.md) → 获取完整内容
3. 最后提炼归档层 (MEMORY.md) → 长期记忆
```

### 检索优化

| 场景 | 策略 |
|------|------|
| 精确查找（已知日期） | 直接读取 memory/YYYY-MM-DD.md |
| 模糊查找（关键词） | 先查索引，再读文件 |
| 主题聚合 | 扫描所有文件，提取相关条目 |

## 缓存策略

### 热点记忆预加载

**触发条件：** 某条目被访问 3 次以上

**动作：**
1. 标记为 `hot: true`
2. 下次 Session 预加载到上下文
3. 7 天无访问→降级为温记忆

### 缓存失效

| 条件 | 动作 |
|------|------|
| 内容更新 | 立即失效缓存 |
| 7 天无访问 | 移出热点缓存 |
| Session 结束 | 清空临时缓存 |

## 核心原则（不忘事/不幻觉）

### 原则 1：不忘事

```
✅ 新会话扫描旧会话
✅ 关键词联系回滚
✅ 记忆连续性保持
✅ 待办事项追踪
```

### 原则 2：不幻觉

```
✅ 只链接已验证记忆
✅ 不编造不存在的内容
✅ 所有链接有来源
✅ 无法验证→标注存疑
```

---

## 新会话启动流程

```
00:00 新会话开始
    ↓
1. 扫描昨日 memory/YYYY-MM-DD.md
    ↓
2. 提取关键词/待办/任务
    ↓
3. 回滚到 HEARTBEAT.md
    ↓
4. 验证记忆链接
    ↓
5. 加载已验证记忆
    ↓
6. 开始新会话
```

---

## 回滚规则（时间分层）

| 规则 | 说明 |
|------|------|
| **7 天内记忆** | 全部回滚 |
| **30 天内记忆** | 关键词匹配回滚 |
| **30 天 + 记忆** | 仅 HEARTBEAT.md |

## 记忆验证机制

### 验证标准

| 验证项 | 标准 |
|--------|------|
| **来源明确** | 有 memory 文件记录 |
| **时间明确** | 有日期/时间戳 |
| **SAYELF 确认** | 有 SAYELF 决策记录 |
| **可验证** | 有外部链接/截图 |

### 标注规则

| 状态 | 标注 | 说明 |
|------|------|------|
| **已验证** | ✅ | 可安全链接 |
| **待验证** | 🟡 | 谨慎链接 |
| **存疑** | ❌ | 不链接 |
