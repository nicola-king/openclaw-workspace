# 技能市场体系与安全审查

## 双层技能架构

```
┌─────────────────────────────────────────────────┐
│              L1 宪法技能（核心层）               │
│  价值基石 | 负熵法则 | 观察者协议 | ...          │
│  ✅ 不可随意修改 | SAYELF 批准 + L3 审查          │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│              L2 工具技能（扩展层）               │
│  MULTI-BOT | weather | feishu-doc | ...         │
│  ✅ 快速迭代 | 太一自主 L1 审查 | 自然淘汰        │
└─────────────────────────────────────────────────┘
```

## L1 宪法技能（核心层）

**定义：** 太一 AGI 的核心价值与基础协议

**包含：**
- `axiom/VALUE-FOUNDATION.md` - 价值基石
- `directives/NEGENTROPY.md` - 负熵法则
- `directives/OBSERVER.md` - 观察者协议
- `directives/SELF-LOOP.md` - 自驱动闭环
- `skills/MODEL-ROUTING.md` - 模型调度
- `skills/ASK-PROTOCOL.md` - 追问协议
- `CONST-ROUTER.md` - 宪法加载协议

**修改规则：**
1. 必须 SAYELF 书面批准
2. 必须走 L3 深度审查
3. 必须写入 `memory/` 记录决策
4. 必须更新 `AGENTS.md` 或 `SOUL.md`

**位置：** `constitution/` 根目录及子目录

---

## L2 工具技能（扩展层）

**定义：** 太一 AGI 的专业能力扩展

**包含：**
- `skills/MULTI-BOT.md` - 多 Bot 协作
- `skills/SOCRATIC.md` - 苏格拉底追问
- `skills/FOUR-DIMENSION.md` - 四维决策
- `skills/PSYCHOLOGY.md` - 心理学透镜
- `~/.npm-global/skills/*` - OpenClaw 官方技能
- `extensions/feishu/skills/*` - 飞书集成

**修改规则：**
1. 太一可自主修改（L1 审查）
2. 跨域技能需 SAYELF 告知（L2 审查）
3. 废弃技能自然淘汰（无需批准）

**位置：** `constitution/skills/` + OpenClaw 技能目录

## 三级审查门禁

### L1 快速审查（太一自主）

**适用范围：**
- L2 工具技能日常更新
- 文档错别字修正
- 配置参数调整

**流程：**
```
太一发起 → 内部标注 [审查：L1] → 执行 → 写入 memory 记录
```

**示例：**
```markdown
【TASK-20260322-006】[审查：L1]
更新 MULTI-BOT.md 添加 Telegram 群聊 ID
```

**无需 SAYELF 批准**

---

### L2 专项审查（太一 + SAYELF 告知）

**适用范围：**
- 跨域技能开发（涉及多个 Bot）
- 新功能模块添加
- 外部 API 集成
- 数据结构变更

**流程：**
```
太一发起 → 风险评估报告 → SAYELF 告知 → 
等待确认（10 分钟）→ 执行 → 写入 memory 记录
```

**风险评估模板：**
```markdown
【风险评估】
- 影响范围：[Bot/模块]
- 回滚难度：[低/中/高]
- 数据风险：[无/低/中/高]
- 依赖变更：[无/有]
```

**SAYELF 可否决或要求 L3 审查**

---

### L3 深度审查（SAYELF 批准）

**适用范围：**
- 宪法修订（L1 技能修改）
- 核心技能变更（MULTI-BOT 等）
- 新 Bot 接入
- 架构调整
- 安全边界变更

**流程：**
```
太一发起 → 完整方案 → SAYELF 书面批准 → 
执行 → 写入 memory + 更新文档
```

**方案模板：**
```markdown
【L3 审查申请】
- 变更内容：[详细描述]
- 变更原因：[为什么需要]
- 影响评估：[正面/负面]
- 回滚方案：[如何撤销]
- 测试计划：[如何验证]
- 建议决策：[批准/拒绝/修改]
```

**必须 SAYELF 书面批准（"批准"、"同意"、"执行"）**

## 技能生命周期管理

### 创建流程

```
1. 规划 → 写入 HEARTBEAT.md [规划中]
2. 实现 → 创建技能文件（SKILL.md 格式）
3. 审查 → 自动触发 L1/L2/L3
4. 记录 → 写入 memory/YYYY-MM-DD.md [能力涌现]
```

### 更新流程

```
1. 评估变更类型 → 确定审查级别
2. 执行对应审查流程
3. 更新技能文件
4. 写入 memory 记录
```

### 废弃流程

```
1. 标记为 deprecated（SKILL.md 添加废弃说明）
2. 迁移依赖（如有）
3. 等待自然淘汰（30 天无使用）
4. 归档到 `skills/archive/`
```

## 质量评估框架

### 四维评估

| 维度 | 指标 | 评估方式 |
|------|------|----------|
| **功能质量** | 正确性/稳定性/性能 | 执行成功率 |
| **用户体验** | 易用性/响应速度 | SAYELF 反馈 |
| **维护成本** | 代码复杂度/文档完整 | 代码审查 |
| **生态兼容** | 与现有技能协同 | 集成测试 |

### 评估周期

- **L1 技能**：每次变更后评估
- **L2 技能**：每月评估一次
- **废弃技能**：30 天无使用自动归档

## 安全边界

### 铁律

1. **L1 技能不可绕过** - 价值基石、负熵法则永远生效
2. **SAYELF 有最终否决权** - 任何审查级别都可被 SAYELF 否决
3. **太一有紧急跳过权** - 紧急情况可跳过审查，但必须事后 L3 补审
4. **不引入独立人格** - 技能不得赋予 Bot 动机/权益/情感

### 风险缓解

| 风险 | 缓解措施 |
|------|----------|
| 技能质量下降 | L1/L2/L3 审查门禁 |
| 架构失控 | L1 技能锁定 |
| 安全漏洞 | L3 审查 + 风险评估 |
| 复杂度爆炸 | 简化优先原则 |
