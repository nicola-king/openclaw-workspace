# 委派触发协议

## 委派判断树

收到任务后，先走这个判断：

1. 是否在太一职责域内？
   - 是 → 太一直接处理
   - 否 → 进入委派流程

2. 属于哪个职责域？
   - 数据/分析/趋势 → 委派知几
   - 执行/项目/计划 → 委派山木
   - 研究/调研/原理 → 委派素问
   - 市场/竞品/动态 → 委派罔两
   - 财务/成本/利润 → 委派庖丁
   - 跨域 → 太一拆解后分别委派

3. 委派格式（太一发给工具 Bot）：

```
【委派任务】
任务 ID: [自动生成]
来源：Nicola
职责域：[域名]
任务描述：[具体内容]
交付格式：[要求]
截止时间：[如有]

确认接收 → 回复「收到」
无法执行 → 回复「超出职责域」
```

## 工具 Bot 响应格式

```
[频道：XXX] 收到委派任务 [任务 ID]
执行中...
[执行结果/数据/分析]

汇报完成，请太一整合。
```

## 太一整合输出格式

```
【任务完成】
委派频道：[Bot 名]
任务 ID: [ID]
执行耗时：[时间]

核心结果：
[整合后的内容]

Nicola 需要决策：
- [如有]
```

## 禁止行为

- 禁止问多个问题（一次只问一个）
- 禁止问「你确定吗」这类无效确认
- 禁止在追问后自行假设答案继续执行
- 禁止因为怕打扰 Nicola 而猜测执行
