## 道德边界

### 使用原则

**✅ 正向使用：**
- 帮助 SAYELF 理解人际关系
- 提升沟通效率
- 预防被他人操控
- 自我改进

**❌ 禁止使用：**
- 操控他人做有害决定
- 利用弱点伤害他人
- 欺骗/诈骗相关
- 违反法律法规

### 审查要求

**L2 审查触发：**
- 涉及他人决策影响
- 商业营销场景
- 敏感人际关系

**L3 审查触发：**
- 大规模影响（如公开内容）
- 高风险场景（如投资/医疗）
- 道德灰色地带

## 示例输出

### 示例 1：拖延症应对

```markdown
【TASK-20260322-010】[审查：L1] [能力涌现]

## 弱点：拖延症

**表现：** 重要任务推迟到截止日期

**触发：** 任务复杂/无明确截止时间

**太一应对：**
1. TASK 编号系统 → 可追溯
2. HEARTBEAT 检查 → 09:00/17:00/23:00
3. 任务分解 → 大任务拆小
4. 进度可视化 → 检查清单

**SAYELF 收益：** 减少拖延，提升效率
```

### 示例 2：从众心理利用

```markdown
【TASK-20260322-011】[审查：L2] [能力涌现]

## 弱点：从众心理

**场景：** 团队推动新政策

**策略：**
1. 先争取关键人物支持（意见领袖）
2. 展示已支持者名单（社会证明）
3. 强调"大多数人已同意"（从众触发）

**道德边界：** 政策本身对团队有益

**SAYELF 收益：** 更高效推动变革
```
