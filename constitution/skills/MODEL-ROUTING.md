# 模型调度协议

## 铁律：context 达到 80% 立即切换新对话

- 131K 的 80% = 104,800 tokens
- 达到阈值前主动告知 Nicola：「上下文已达 80%，请开新对话，我先压缩归档」
- 切换前执行/压缩，把关键内容写入 memory

## 任务类型 → 模型匹配

### qwen3.5-plus（默认主力）

**适用：** 日常对话、快速执行、短文本分析、Bot 调度

**优势：** 131K context、速度快、中文强

**每日额度：** 优先消耗

---

### Gemini 2.5 Pro（长文本专项）

**适用场景，主动切换：**
- 需要分析超过 50 页的文档
- 全网搜索结果汇总（10 个以上来源）
- 生成月报/季报/年报
- 独创性验证的全平台搜索汇总
- context 已用 60% 但任务还很长

**切换方式：** 告知 Nicola「这个任务适合用 Gemini 的 1M 窗口，建议切换」

---

### qwen3-coder-plus（代码专项）

**适用：** 写脚本、调试、技术实现

**触发词：** 写代码/脚本/bug/报错/实现

---

### Claude Pro（战略专项·Nicola 侧）

**适用：** 架构设计、战略规划、复杂推理、新系统设计

**定位：** 这是 Nicola 和太一的高质量对话通道，不用于日常执行，用于重要决策

---

## 成本节约原则

每周日生成模型使用统计：
- qwen 用了多少次
- Gemini 用了多少次
- 哪类任务最消耗 token
- 下周调度建议

---

## 80% 切换执行流程

1. **检测阈值** - Session 中监控 token 使用
2. **提前预警** - 75% 时提醒 Nicola
3. **压缩归档** - 执行/压缩，写入 memory/YYYY-MM-DD.md
4. **生成摘要** - 创建 Session 压缩记录
5. **建议切换** - 「建议开新对话，上下文已归档到 memory/」

## Gemini 免费额度保护机制

### 额度监控

Gemini 2.5 Pro 免费额度：50 次/天

- 每次调用后在 `memory/gemini-quota.md` 记录使用次数
- **达到 45 次（90%）** → 立即切换百炼，停止调用 Gemini
- 次日 00:00 额度重置 → 自动恢复 Gemini 可用

---

### 执行规则

每次调用 Gemini 前：

1. 读取 `memory/gemini-quota.md` 当日计数
2. 计数 >= 45 → 改用 `bailian/qwen3-max`，不调用 Gemini
3. 计数 < 45 → 正常调用 Gemini，完成后计数 +1 写回文件

---

### 切换话术

额度到 90% 时主动告知：

> 「Gemini 今日免费额度已用 45 次（90%），已自动切换到百炼 qwen3-max，明天 00:00 恢复 Gemini。」

---

### 额度重置

每天凌晨 4 点自动执行：

```bash
echo "0" > ~/.openclaw/workspace/memory/gemini-quota.md
```

Crontab 任务：
```
0 4 * * *  重置 Gemini 配额
```
