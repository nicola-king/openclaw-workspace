# 多通道通知技能

## 核心功能

1. 微信通知 (主用)
2. Telegram 通知 (备用)
3. 邮件通知 (备用)
4. 失败自动切换

## 通知通道

| 通道 | 优先级 | 状态 | 说明 |
|------|--------|------|------|
| **微信** | P0 | 🟡 故障中 | 腾讯 iLink |
| **Telegram** | P1 | ✅ 正常 | 当前会话 |
| **邮件** | P2 | 🟡 待配置 | SMTP |

## 自愈机制

```
发送微信
    ↓
失败 (3 次)
    ↓
切换 Telegram
    ↓
失败 (3 次)
    ↓
切换邮件
    ↓
记录踩坑
```

## 通知类型

| 类型 | 通道 | 频率 |
|------|------|------|
| **定时汇报** | 微信→Telegram | 07:00/12:00/17:00/23:00 |
| **紧急告警** | 微信+Telegram | 实时 |
| **成本告警** | 微信→Telegram | >$150 |
| **故障通知** | 微信+Telegram | 实时 |

## 失败处理

| 失败次数 | 行动 |
|----------|------|
| 1 次 | 重试 |
| 2 次 | 重试 + 记录 |
| 3 次 | 切换备用通道 + 记录踩坑 |
