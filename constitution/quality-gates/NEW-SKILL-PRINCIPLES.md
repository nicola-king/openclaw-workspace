## 🔍 P1: 蒸馏来源

**核心问题：** 这个 Skills/Agent/软件从哪里来？

**执行清单：**
- [ ] 来源链接/仓库地址
- [ ] 作者/维护者信息
- [ ] 许可证类型（MIT/Apache/GPL/...）
- [ ] 社区活跃度（Stars/Forks/Issues）
- [ ] 最后更新时间

**拒绝标准：**
- ❌ 来源不明
- ❌ 无许可证
- ❌ 已废弃（>1 年未更新）
- ❌ 恶意作者（有不良记录）

**输出：**
```markdown
【来源蒸馏】
- 原始链接：
- 作者/维护者：
- 许可证：
- 社区数据：Stars/Forks/Issues
- 最后更新：
- 蒸馏理由：
```

## 🎯 P4: 取其精华（不重复覆盖）

**核心问题：** 只增加提升部分，不重复造轮子

**执行策略：**

### 策略 A：替换（新技能全面优于现有）
```
条件：新技能在 80%+ 维度优于现有
执行：
  1. 备份现有技能
  2. 安装新技能
  3. 迁移配置
  4. 验证功能
  5. 删除备份
```

### 策略 B：融合（各有优势）
```
条件：新技能与现有技能互补
执行：
  1. 保留现有技能核心
  2. 提取新技能优势模块
  3. 创建融合版本
  4. 标注来源
```

### 策略 C：并列（不同场景使用）
```
条件：新技能适用于不同场景
执行：
  1. 安装新技能
  2. 明确使用场景边界
  3. 更新文档说明
```

### 策略 D：拒绝（无提升）
```
条件：新技能无显著提升
执行：
  1. 记录拒绝理由
  2. 写入 memory/
  3. 不安装
```

**输出：**
```markdown
【决策】
- 策略：A 替换 / B 融合 / C 并列 / D 拒绝
- 理由：
- 执行计划：
```

## 🔒 P6: 安全评估

**核心问题：** 这个技能/软件安全吗？

**评估维度：**

### 代码安全
- [ ] 无恶意代码（后门/木马）
- [ ] 无敏感信息泄露（硬编码密码）
- [ ] 无危险操作（rm -rf / chmod 777）
- [ ] 依赖安全（无已知漏洞）

### 权限安全
- [ ] 最小权限原则
- [ ] 不需要 root/sudo
- [ ] 不修改系统文件
- [ ] 不监听敏感端口

### 数据安全
- [ ] 不上传用户数据
- [ ] 不收集隐私信息
- [ ] 本地存储加密
- [ ] 符合 GDPR/隐私政策

**命令参考：**
```bash
# 扫描依赖漏洞
pip-audit
npm audit

# 检查危险操作
grep -r "rm -rf\|chmod 777\|eval(" 技能目录

# 检查网络请求
grep -r "requests.post\|urllib.request" 技能目录
```

**输出：**
```markdown
【安全评估】
- 代码安全：✅ 通过 / ⚠️ 警告 / ❌ 拒绝
- 权限安全：✅ 通过 / ⚠️ 警告 / ❌ 拒绝
- 数据安全：✅ 通过 / ⚠️ 警告 / ❌ 拒绝
- 依赖漏洞：0 个高危 / 0 个中危
- 总体评级：A/B/C/D
```

## 🛡️ P7: 系统保护

**核心问题：** 安装后会对系统造成伤害吗？

**保护清单：**

### 安装前
- [ ] 创建系统快照/备份
- [ ] 准备回滚方案
- [ ] 测试环境验证（如可能）

### 安装中
- [ ] 使用虚拟环境（venv/conda）
- [ ] 不修改全局配置
- [ ] 记录所有变更

### 安装后
- [ ] 验证系统稳定性
- [ ] 监控资源占用（CPU/内存/磁盘）
- [ ] 检查日志无异常
- [ ] 验证现有功能不受影响

**回滚方案：**
```bash
# 备份
cp -r skills/xxx skills/xxx.backup.$(date +%Y%m%d)

# 回滚
rm -rf skills/xxx
mv skills/xxx.backup.* skills/xxx

# 恢复服务
systemctl restart openclaw-gateway
```

**输出：**
```markdown
【系统保护】
- 备份完成：✅
- 回滚方案：✅ 已准备
- 资源影响：CPU +0% / 内存 +0MB / 磁盘 +0MB
- 现有功能：✅ 无影响
- 监控状态：✅ 正常
```

## 🧬 P8: 融合·递归·进化·涌现

**核心问题：** 如何让新技能产生 1+1>2 的效果？

### 融合（Integration）
**目标：** 新技能与现有系统无缝集成

**执行：**
- [ ] API 接口对接
- [ ] 数据格式统一
- [ ] 配置集中管理
- [ ] 日志统一格式

### 递归（Recursion）
**目标：** 技能可以自我改进

**执行：**
- [ ] 添加自我监控
- [ ] 性能自动优化
- [ ] 错误自动修复
- [ ] 版本自动更新

### 进化（Evolution）
**目标：** 技能随使用而变强

**执行：**
- [ ] 收集使用反馈
- [ ] 记录最佳实践
- [ ] 定期迭代升级
- [ ] 社区贡献机制

### 涌现（Emergence）
**目标：** 多技能协作产生新能力

**执行：**
- [ ] 技能间通信机制
- [ ] 跨技能工作流
- [ ] 组合技能包
- [ ] 能力涌现记录

**输出：**
```markdown
【进化计划】
- 融合点：与 [技能 A] + [技能 B] 集成
- 递归机制：自我监控 + 自动优化
- 进化路径：v1.0 → v2.0（3 个月）
- 涌现潜力：与 [技能 C] 组合可产生 [新能力]
```
