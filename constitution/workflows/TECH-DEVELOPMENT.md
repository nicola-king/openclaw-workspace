## 📋 执行流程

### 阶段 1: 需求分析
- [ ] 理解核心需求（5 Why 分析）
- [ ] 确认技术栈
- [ ] 确认输入/输出
- [ ] 确认边界条件
- [ ] 确认验收标准

### 阶段 2: 方案设计
- [ ] 架构设计（模块化）
- [ ] 接口定义
- [ ] 数据流设计
- [ ] 错误处理设计
- [ ] 回滚机制设计

### 阶段 3: 安全评估
- [ ] 风险评估（INSTALL-PROTOCOL.md）
- [ ] 权限评估（最小权限原则）
- [ ] 依赖评估（官方源优先）
- [ ] 备份方案设计

### 阶段 4: 实现开发
- [ ] 代码实现（注释齐全）
- [ ] 单元测试
- [ ] 集成测试
- [ ] 文档编写

### 阶段 5: 部署验证
- [ ] 部署到测试环境
- [ ] 功能验证
- [ ] 性能验证
- [ ] 回滚测试
- [ ] 生产部署（如需要）

## 📤 输出规范

### 代码规范
| 要求 | 说明 |
|------|------|
| 注释 | 关键逻辑必须有注释 |
| 命名 | 语义化命名 |
| 错误处理 | 必须有 try-catch 或等效 |
| 日志 | 关键节点必须有日志 |
| 配置 | 敏感信息用环境变量 |

### 文档规范
| 文档 | 说明 |
|------|------|
| README.md | 项目说明 + 快速开始 |
| API.md | 接口文档（如有） |
| DEPLOY.md | 部署指南 |
| TROUBLESHOOTING.md | 故障排查 |
