## 🎯 核心原则

```
学习必须深度 · 按顺序逐张阅读 · 立即秩序执行
```

---

## 📋 8 步深度学习法 + 第 9 原则

### 1️⃣ 按顺序逐张阅读
- [ ] 按收到顺序编号
- [ ] 逐张阅读，不跳过
- [ ] 标记已读状态
- [ ] 记录未读数量

### 2️⃣ 验证真实性原文
- [ ] 找到原始来源（GitHub/官网）
- [ ] 验证作者身份
- [ ] 检查发布时间
- [ ] 查看社区反馈
- [ ] 交叉验证

### 3️⃣ 蒸馏
- [ ] 一句话概括核心内容
- [ ] 提取 3-5 个关键要点
- [ ] 删除营销/夸张表述
- [ ] 保留数据/事实/参数

### 4️⃣ 本地系统穿透
- [ ] 太一是否有类似功能？
- [ ] 太一现状如何？
- [ ] 差距在哪里？
- [ ] 是否需要跟进？

### 5️⃣ 比对
- [ ] 对方优势是什么？
- [ ] 对方劣势是什么？
- [ ] 太一优势是什么？
- [ ] 太一劣势是什么？
- [ ] 是否值得学习？

### 6️⃣ 提炼精华去糟粕
- [ ] 哪些是精华（必须学习）？
- [ ] 哪些是糟粕（应该放弃）？
- [ ] 哪些需要改良（适配太一）？

### 7️⃣ 验证安全性稳定性可靠性
- [ ] 安全性评估
- [ ] 稳定性评估
- [ ] 可靠性评估
- [ ] 成本评估
- [ ] 风险评估

### 8️⃣ 融合
- [ ] 写入记忆（memory/YYYY-MM-DD.md）
- [ ] 更新相关技能/配置
- [ ] 创建行动清单
- [ ] 设定执行时间
- [ ] 8 步流程上报

### 9️⃣ 立即秩序执行 ⚡
- [ ] 检查融合方案是否存在
- [ ] 检查行动计划是否明确
- [ ] 检查执行时间是否具体
- [ ] 立即执行/本周执行/下周执行
- [ ] 执行结果上报
