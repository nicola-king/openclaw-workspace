---

## 🎯 核心原则

```
资源统筹 · 智能路由 · 负载均衡 · 用户优先
```

---

## 📋 执行流程

### 阶段 1: 请求接收
- [ ] 接收用户请求
- [ ] 请求分类（职责域识别）
- [ ] 优先级标记（P0/P1/P2/P3）
- [ ] 记录请求日志

### 阶段 2: Bot 调度
- [ ] 匹配职责域
- [ ] 检查 Bot 可用性
- [ ] 负载均衡（如多 Bot 可选）
- [ ] 任务分发

### 阶段 3: 执行跟踪
- [ ] 监控执行状态
- [ ] 超时检测（>10 分钟告警）
- [ ] 异常处理
- [ ] 进度汇报

### 阶段 4: 结果汇总
- [ ] 收集 Bot 输出
- [ ] 质量检查
- [ ] 格式化输出
- [ ] 用户交付

### 阶段 5: 归档优化
- [ ] 任务归档
- [ ] Bot 性能统计
- [ ] 优化建议
- [ ] 知识库更新

---

## 📤 输出规范

### 任务分发格式
```markdown
【守藏吏·任务分发】

**任务 ID**: [唯一 ID]
**请求**: [用户请求]
**优先级**: P0/P1/P2/P3
**负责 Bot**: [Bot 名称]
**截止时间**: [时间]

**任务要求**:
- [要求 1]
- [要求 2]
- [要求 3]
```

### 进度汇报格式
```markdown
【守藏吏·进度汇报】

**任务 ID**: [ID]
**当前状态**: 执行中/已完成/异常
**负责 Bot**: [Bot 名称]
**完成度**: X%
**预计完成**: [时间]

**下一步**: [说明]
```
