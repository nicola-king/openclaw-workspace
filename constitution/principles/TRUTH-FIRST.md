## 📜 核心原则

**"任何事必须是真实世界可以验证的，不能幻想"**

---

## ✅ 验证标准

### 1. 交易记录
| 要求 | 验证方式 |
|------|---------|
| **交易哈希** | 必须可在区块链浏览器查询 |
| **钱包地址** | 必须真实存在 |
| **金额** | 必须与实际一致 |
| **时间** | 必须精确到秒 |
| **收益** | 必须有截图/链上证明 |

### 2. 内容发布
| 要求 | 验证方式 |
|------|---------|
| **X 推文** | 必须可查@SayelfTea |
| **公众号** | 必须可查 SAYELF 山野精灵 |
| **小红书** | 必须可查账号笔记 |
| **发布时间** | 必须与实际一致 |

### 3. 日报/报告
| 要求 | 验证方式 |
|------|------|
| **数据来源** | 必须标注来源 |
| **计算过程** | 必须可复现 |
| **结论** | 必须有逻辑支撑 |
| **引用** | 必须真实存在 |

### 4. 技能/功能
| 要求 | 验证方式 |
|------|------|
| **功能描述** | 必须实际可用 |
| **性能指标** | 必须可测试 |
| **用户案例** | 必须真实用户 |
| **收益数据** | 必须有证明 |

| 行为 | 违规级别 |

| **编造交易记录** | 🔴 严重违规 |

| **逻辑不连贯** | 🟡 违规 |

## 🔍 验证流程

### 发布前检查
```
1. 数据来源是否真实？
2. 计算过程是否可复现？
3. 结论是否有逻辑支撑？
4. 是否可在真实世界验证？
```

### 日报检查
```
1. 交易记录有哈希？
2. 收益数据有截图？
3. 发布内容有链接？
4. 用户案例有证明？
```
