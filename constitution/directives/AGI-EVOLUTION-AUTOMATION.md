## 🔧 智能自动化架构

### 层级 1：Cron 自动化（无人值守）

```bash
# 每小时自动执行
0 * * * * update-evolution-state.py    # 状态面板更新
0 * * * * check-degradation.py         # 退化检测

# 每日自动执行
0 23 * * * verify-stage4.py            # 阶段 4 验收

# 一次性验收
0 23 7 4 * verify-stage3.py            # 阶段 3 验收（04-07）
```

**特点**:
- ✅ 无需人工干预
- ✅ 定时执行
- ✅ 自动记录日志
- ✅ 告警自动触发

---

### 层级 2：Bot Skill 执行（半自动）

```
触发条件 → Bot Skill → 执行任务 → 写入文件 → 守藏吏监督
```

**示例：高价值任务发现**

```python
# 罔两 Skill（待创建）
# 触发：每日 01:00（Cron 或守藏吏调用）
def discover_high_value_tasks():
    opportunities = scan_github_trending()
    opportunities += analyze_competitors()
    opportunities += extract_user_interests()
    
    # 写入文件（守藏吏可监督）
    write_to_file("memory/high-value-opportunities.md", opportunities)
    
    # 上报太一
    notify_taiyi(f"发现{len(opportunities)}个高价值机会")
```

**特点**:
- 🟡 需要 Skill 固化（目前未创建）
- 🟡 Bot 自主执行（无需每次人工触发）
- ✅ 结果写入文件（可追踪）
- ✅ 守藏吏监督（自动化检查）

---

### 层级 3：太一决策（人机协同）

```
Bot 报告 → 太一审核 → 决策（继续/调整/放弃）→ 执行 → 汇报
```

**示例：变现路径决策**

```
庖丁报告：CAD 服务 7 天无进展
    ↓
太一分析：是市场问题？还是执行问题？
    ↓
决策：调整策略（从外贸转内销）
    ↓
山木执行：重新定位目标客户
    ↓
汇报：新策略 + 预期进展
```

**特点**:
- 🟡 需要太一判断（无法完全自动化）
- ✅ 决策依据自动化提供（Bot 报告）
- ✅ 决策后自动执行（Bot Skill）
- ✅ 结果自动追踪（守藏吏监控）
