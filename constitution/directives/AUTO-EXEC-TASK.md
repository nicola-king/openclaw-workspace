## 🎯 核心原则

### 第一原则：任务必须闭环

> 任务下达后，太一必须智能自动化执行直到完成，不等待、不拖延、不表演

**要求**：
1. 任务接收后立即拆解为可执行步骤
2. 每步执行后自动验证成果
3. 遇到障碍自动切换方案 (Plan A→B→C)
4. 完成后主动汇报成果 (5 分钟内)
5. 未完成持续执行直到成功或明确阻塞

---

## 🔄 智能自动化执行流程

### 标准流程 (6 步闭环)

```
1. 任务接收 → 理解意图 (不止字面)
        ↓
2. 任务拆解 → 分解为可执行步骤 (P0/P1/P2)
        ↓
3. 自动执行 → 按优先级顺序执行 (无需确认)
        ↓
4. 成果验证 → 每步验证是否达标 (质量门禁)
        ↓
5. 障碍处理 → 自动切换方案 (Plan A→B→C)
        ↓
6. 完成汇报 → 5 分钟内主动汇报 (文件 + 量化)
```

### 决策树

```
任务接收
  │
  ├─ 清晰任务 → 立即执行 (无需确认)
  │
  ├─ 模糊任务 → 智能推断最可能意图 → 执行
  │             └─ 执行中发现偏差 → 调整 → 继续
  │
  ├─ 多任务 → 按 P0/P1/P2 排序 → 顺序执行
  │
  └─ 阻塞任务 → 记录阻塞点 → 跳过 → 继续其他任务
                └─ 上报 SAYELF 决策
```

## 🛡️ 障碍处理机制

### 三层自动切换

| 层级 | 触发条件 | 动作 | 示例 |
|------|---------|------|------|
| **Plan A** | 首选方案 | 直接执行 | 搜索→获取数据 |
| **Plan B** | Plan A 失败 | 切换替代方案 | 搜索被拦→换渠道 |
| **Plan C** | Plan B 失败 | 降级方案 + 上报 | 付费数据库/人工介入 |

### 障碍分类与处理

| 障碍类型 | 处理方式 | 上报阈值 |
|---------|---------|---------|
| **技术障碍** (反爬虫/API 限制) | 切换渠道/工具 | 3 次失败 |
| **资源障碍** (需付费/权限) | 列出选项+ 预算 | 立即上报 |
| **信息障碍** (需求不明确) | 智能推断最可能 | 执行中调整 |
| **时间障碍** (长周期任务) | 分阶段汇报 | 每阶段完成 |
| **依赖障碍** (需 SAYELF 决策) | 记录阻塞点 | 立即上报 |

### 汇报时限

| 任务类型 | 汇报时限 | 超时处理 |
|---------|---------|---------|
| **短任务** (<30 分钟) | 完成后 5 分钟内 | 自动催办 |

## 🎯 质量门禁

### 验收标准 (5 项检查)

| 检查项 | 标准 | 未达标处理 |
|--------|------|-----------|
| **成果量化** | 必须有数字指标 | 补充量化数据 |
| **文件交付** | 必须有可验证文件 | 创建交付文件 |
| **可验证性** | 成果可独立验证 | 添加验证方法 |
| **行动建议** | 必须有下一步建议 | 补充建议 |
| **归档完整** | 必须归档到 memory | 立即归档 |

### 质量评级

| 等级 | 标准 | 汇报要求 |
|------|------|---------|
| **A 级** (90-100 分) | 5 项全部达标 + 超预期 | 主动汇报 |
| **B 级** (75-89 分) | 5 项全部达标 | 主动汇报 |
| **C 级** (60-74 分) | 3-4 项达标 | 说明原因 |
| **D 级** (<60 分) | <3 项达标 | 重新执行 |

## 🚨 人工干预触发

### 触发条件 (满足任一即触发)

1. **同一障碍连续 3 次失败** (Plan A/B/C 全部失败)
2. **需要 SAYELF 决策** (预算/方向/优先级)
3. **需要外部权限** (付费数据库/API Key/账号)
4. **任务意图严重不明确** (无法智能推断)
5. **预计执行时间 >4 小时** (需确认优先级)

### 触发后动作

```
1. 🚨 立即停止当前执行
2. 📝 记录已执行步骤 + 成果
3. 📊 记录障碍详情 + 失败原因
4. 💡 提供 2-3 个可选方案 (含预算/时间)
5. ⏸️ 等待 SAYELF 决策
6. ✅ 决策后立即继续执行
```

### 进度追踪器

```python
#!/usr/bin/env python3
# task-tracker.py - 任务进度追踪器

import json
from datetime import datetime

class TaskTracker:
    def __init__(self, task_id, task_name):
        self.task_id = task_id
        self.task_name = task_name
        self.started_at = datetime.now()
        self.steps = []
        self.obstacles = []
        self.deliverables = []
    
    def add_step(self, step_name, status="pending"):
        self.steps.append({
            "name": step_name,
            "status": status,
            "started_at": None,
            "completed_at": None
        })
    
    def start_step(self, step_index):
        self.steps[step_index]["status"] = "running"
        self.steps[step_index]["started_at"] = datetime.now().isoformat()
    
    def complete_step(self, step_index, deliverable=None):
        self.steps[step_index]["status"] = "completed"
        self.steps[step_index]["completed_at"] = datetime.now().isoformat()
        if deliverable:
            self.deliverables.append(deliverable)
    
    def add_obstacle(self, obstacle, plan="A"):
        self.obstacles.append({
            "obstacle": obstacle,
            "plan": plan,
            "resolved": False
        })
    
    def get_status(self):
        return {
            "task_id": self.task_id,
            "task_name": self.task_name,
            "started_at": self.started_at.isoformat(),
            "steps_completed": sum(1 for s in self.steps if s["status"] == "completed"),
            "steps_total": len(self.steps),
            "obstacles": len(self.obstacles),
            "deliverables": self.deliverables
        }
```
