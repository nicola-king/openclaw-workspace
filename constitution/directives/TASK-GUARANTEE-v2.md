# 定时任务保障法则（Task Guarantee Protocol）

> 版本：v2.0 | 创建时间：2026-04-05 00:55 | 级别：Tier 1 永久核心

---

## 🎯 核心原则

> **文件 > 记忆 · 自动化 > 手动 · 冗余 > 单一 · 验证 > 假设**

定时任务是系统自主运行的核心保障，必须实现：
1. **零遗忘** - 任务执行记录永久保存
2. **零漏报** - 执行失败立即告警
3. **零中断** - 故障自动恢复
4. **可验证** - 每次执行可追溯

---

## 📋 三层防护机制

### 第一层：执行监控（实时）

**职责：** 监控每次任务执行，记录状态

**实现：**
- OpenClaw 内置任务执行日志 (`~/.openclaw/cron/runs/*.jsonl`)
- 执行状态实时写入 (`/tmp/task-exec-status.json`)
- 失败立即触发告警

**配置：**
```json
{
  "monitoring": {
    "enabled": true,
    "checkInterval": "*/5 * * * *",
    "alertThreshold": 1,
    "autoRecovery": true
  }
}
```

---

### 第二层：健康检查（每小时）

**职责：** 检查任务执行状态，发现异常

**实现：**
- 每小时检查所有任务最后执行时间
- 超过预期时间未执行 → 触发告警
- 连续错误 > 3 次 → 自动修复

**脚本：** `scripts/task-health-check.sh`

**检查项：**
```bash
1. 检查任务是否启用
2. 检查最后执行时间（是否超时）
3. 检查错误计数（是否超限）
4. 检查通道状态（是否正常）
5. 检查执行日志（是否有输出）
```

---

### 第三层：自愈恢复（自动）

**职责：** 自动修复故障任务

**触发条件：**
- 任务连续错误 ≥ 3 次
- 任务超时未执行 ≥ 2 小时
- 通道状态异常

**自愈动作：**
```bash
1. 重置错误计数
2. 切换备用通道（weixin → telegram → feishu）
3. 重启 Gateway（如必要）
4. 发送告警通知
5. 记录修复日志
```

---

## 🔔 告警机制

### 告警级别

| 级别 | 条件 | 响应时间 | 通知方式 |
|------|------|---------|---------|
| **P0 紧急** | 所有任务失败 | 5 分钟 | Telegram + 微信 |
| **P1 严重** | ≥3 个任务失败 | 15 分钟 | Telegram |
| **P2 警告** | 1-2 个任务失败 | 1 小时 | Telegram |
| **P3 提示** | 任务延迟执行 | 4 小时 | 日报汇总 |

### 告警内容

```
【定时任务告警 · P1 严重】

时间：2026-04-05 07:05
级别：P1 严重

故障任务：
1. weather-lite-daily - 连续错误 3 次
2. zhiji-weather - 连续错误 3 次
3. shanmu-morning - 超时未执行

影响：气象数据采集可能中断

自愈动作：
✅ 已重置错误计数
✅ 已切换通道 (weixin → telegram)
🟡 等待下次执行验证

详情：/home/nicola/.openclaw/workspace/reports/task-alert-20260405.md
```

## 📊 验证机制

### 每日验证（06:00）

**脚本：** `scripts/daily-task-verify.sh`

**检查项：**
```bash
1. 检查昨日所有任务是否执行
2. 检查执行成功率（目标：100%）
3. 检查告警次数（目标：0）
4. 检查自愈成功率（目标：100%）
5. 生成日报
```

**输出：** `reports/task-daily-report-YYYYMMDD.md`

---

### 每周验证（周一 06:00）

**脚本：** `scripts/weekly-task-audit.sh`

**检查项：**
```bash
1. 汇总本周所有任务执行记录
2. 分析失败原因
3. 优化任务配置
4. 更新 HEARTBEAT.md
5. 生成周报
```

**输出：** `reports/task-weekly-report-YYYYMMDD.md`
