# 太一任务保障法则

> 版本：v1.0 | 创建：2026-03-29 22:15 | 级别：宪法级

---

## 🎯 核心原则

```
文件 > 记忆 · 自动化 > 手动 · 冗余 > 单一
```

**铁律**: 任何任务必须经过文件化，不依赖对话记忆。

---

## 🔒 五层防护机制

### 第 1 层：文件化存储（永久）

**原则**: 任务必须写入文件，不依赖对话记忆

**执行**:
- 位置：`constitution/tasks/TASK-*.md`
- 格式：标准化任务模板
- 备份：每日记忆归档自动包含

**验证**: 会话重启后，任务文件依然存在

---

### 第 2 层：定时任务检查（每日 06:00）

**原则**: 每天自动检查任务进度

**执行**:
- 脚本：`scripts/daily-constitution.sh`
- 时间：每日 06:00（宪法学习时）
- 内容：读取任务文件 + 更新 HEARTBEAT.md

**验证**: 每次 06:00 执行后，日志有记录

---

### 第 3 层：任务监控脚本（每小时）

**原则**: 每小时检查，及时发现风险

**执行**:
- 脚本：`scripts/task-monitor.sh`
- 频率：每小时自动运行
- 内容：当日截止 + 逾期任务 + 即将到来截止

**验证**: `logs/task-monitor.log` 每小时有记录

---

### 第 4 层：HEARTBEAT 待办（每次心跳）

**原则**: 核心待办常驻显示

**执行**:
- 位置：`HEARTBEAT.md` - 核心待办
- 更新：每日 06:00 自动更新
- 显示：P0 任务置顶

**验证**: 查看 HEARTBEAT.md，任务在待办列表中

---

### 第 5 层：日报生成（每日 23:00）

**原则**: 每日复盘，确保连续性

**执行**:
- 时间：每日 23:00
- 内容：任务进度汇总 + 次日预告
- 归档：写入 memory/YYYY-MM-DD.md

**验证**: 每日日报包含任务进度

---

## 🌙 凌晨时段优化策略

### 核心洞察

**观察**: 凌晨时段（02:00-06:00）是大模型使用最少的时间段

**优势**:
- API 请求少，响应更快
- 网络负载低，稳定性高
- 无用户干扰，可批量执行
- 资源竞争少，成本更低

---

### 凌晨时段任务分配

| 时间 | 任务 | 负责 | 说明 |
|------|------|------|------|
| **02:00** | 数据采集批量任务 | 罔两 | 网页爬虫/API 调用 |
| **03:00** | 内容批量生成 | 山木 | 文章/帖子草稿 |
| **04:00** | 代码开发/测试 | 素问 | 技能开发/测试 |
| **05:00** | 数据备份/整理 | 守藏吏 | 记忆归档/备份 |
| **06:00** | 宪法学习 + 任务检查 | 太一 | 每日例行 |

---

### 凌晨时段执行原则

**优先级**:
1. P0: 批量数据采集（网络负载低）
2. P1: 批量内容生成（无干扰）
3. P2: 技能开发/测试（资源充足）
4. P3: 数据备份/整理（系统空闲）

**限制**:
- 不执行实时交易任务（需人工监督）
- 不执行高风险操作（需即时响应）
- 不执行用户交互任务（用户在休息）

---

### 凌晨时段任务调度

**自动触发条件**:
```
时间：02:00-06:00
任务类型：批量/离线/低风险
资源：API 配额充足
```

**执行流程**:
```
02:00 检查任务队列
    ↓
筛选适合凌晨执行的任务
    ↓
批量执行（最大化利用低负载）
    ↓
记录执行日志
    ↓
06:00 汇报执行结果
```

## 📊 任务分类与执行时机

| 任务类型 | 最佳时段 | 原因 |
|---------|---------|------|
| 批量数据采集 | 02:00-03:00 | 网络负载低 |
| 批量内容生成 | 03:00-04:00 | 无用户干扰 |
| 技能开发/测试 | 04:00-05:00 | 资源充足 |
| 数据备份/归档 | 05:00-06:00 | 系统空闲 |
| 实时交易 | 白天 | 需人工监督 |
| 用户交互 | 白天 | 用户活跃 |
| 紧急任务 | 立即 | 不分时段 |

## 🚨 任务执行保障

### 失败处理

| 情况 | 处理 |
|------|------|
| 凌晨任务失败 | 记录日志 + 06:00 汇报 |
| API 配额不足 | 暂停 + 等待下一周期 |
| 系统异常 | 告警 + 太一介入 |
| 任务逾期 | 🔴 P0 告警 + 通知 SAYELF |

### 成功验证

| 检查项 | 验证方式 |
|--------|---------|
| 文件化 | 检查任务文件存在 |
| 定时执行 | 检查日志时间戳 |
| 监控运行 | 检查 task-monitor.log |
| HEARTBEAT | 检查待办列表 |
| 日报生成 | 检查日报内容 |
