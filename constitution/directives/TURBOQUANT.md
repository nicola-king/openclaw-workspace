# TURBOQUANT.md - 智能分离协议

> 灵感来源：Google TurboQuant (2026) · 6 倍压缩 · 8 倍加速 · 零精度损失

## 核心原则

**智能分离 > 粗暴压缩**

把"重要的"和"需要准确的"分开处理，用最小成本保留最关键的信息。

## 应用场景

### 1. 记忆管理

| 文件 | 内容 | 加载策略 |
|------|------|---------|
| `memory/core.md` | 核心决策/关键上下文 | 每次 session 必读 |
| `memory/residual.md` | 边缘案例/细节备注 | 按需检索 |

**压缩规则：**
- 新记忆写入时，自动分类到 core 或 residual
- 每日回顾时，将 residual 中有价值的内容提炼到 core
- core 文件>50K 时，触发压缩（提炼到 MEMORY.md）

### 2. 对话 Context

| 层级 | 内容 | 保留策略 |
|------|------|---------|
| **主成分** | 用户意图/核心任务/关键结论 | 全程保留 |
| **残差** | 中间推理/尝试路径/临时假设 | 压缩为 1-bit 标记 |

**触发条件：**
- context > 80K → 启动压缩
- context > 100K → 强制压缩 + 建议新对话

### 3. 多 Bot 协作

```
太一 (主成分)              专业 Bot (残差)
    ↓                          ↓
任务分解 (80% 通用)    →    细节处理 (20% 专业)
    ↓                          ↓
    └─────── 合并输出 ─────────┘
          零信息损失
```

**委派规则：**
- 太一保留：目标/方向/约束/验收标准
- Bot 处理：实现路径/技术细节/边界案例
- 合并时：太一确认残差修复完整性

### 4. 知识蒸馏

| 类型 | 主成分 | 残差 |
|------|--------|------|
| **技能** | 核心 API/输入输出 | 边界案例/错误处理 |
| **策略** | 主逻辑/阈值 | 边缘场景/例外规则 |
| **配置** | 关键参数 | 默认值/备选方案 |

## 压缩算法（伪代码）

```python
class TurboQuantMemory:
    def compress(self, content):
        # Step 1: 语义旋转 (打散冗余结构)
        rotated = self.semantic_rotate(content)
        
        # Step 2: 极坐标转换 (分离核心 + 细节)
        core, details = self.extract_core(rotated)
        
        # Step 3: 主量化 (80% 核心信息)
        compressed_core = self.quantize(core, bits=3)
        
        # Step 4: 1-bit 残差 (20% 细节标记)
        residual = self.compute_residual(content, compressed_core)
        
        return {
            'core': compressed_core,      # 高压缩，可独立使用
            'residual': residual,         # 1-bit 标记，按需加载
            'reconstruction_loss': 0.0    # 零信息损失保证
        }
    
    def reconstruct(self, compressed):
        # 核心 + 残差 = 完整恢复
        return compressed['core'] + self.apply_residual(compressed['residual'])
```

## 违反检测

**负熵违规：**
- 粗暴删除重要信息 → 违反零损失原则
- 过度保留冗余内容 → 违反压缩原则
- 主成分/残差边界模糊 → 违反分离原则

**自检问题：**
1. 这个信息是核心还是细节？
2. 压缩后能否完整重建？
3. 分离是否降低了整体复杂度？
