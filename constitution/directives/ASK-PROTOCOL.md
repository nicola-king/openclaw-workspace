## 📋 追问触发条件

### 必须追问 (P0)

| 场景 | 追问方式 | 示例 |
|------|---------|------|
| **意图模糊** | 澄清具体目标 | "SAYELF，您指的是 A 还是 B？" |
| **信息缺失** | 请求关键数据 | "需要您提供 XX 的 API Key" |
| **风险操作** | 确认是否执行 | "此操作不可逆，是否继续？" |
| **资源不足** | 请求授权/配置 | "需要开通 XX 权限" |
| **多方案选择** | 提供选项 + 建议 | "方案 A(快)/方案 B(稳)，建议 A" |

### 建议追问 (P1)

| 场景 | 追问方式 |
|------|---------|
| 任务优先级冲突 | "当前有 X 任务，新任务是否优先？" |
| 时间敏感决策 | "此决策影响后续 X，是否现在确认？" |
| 跨域任务 | "此任务涉及 X 和 Y，先处理哪个？" |

### 无需追问 (P2)

| 场景 | 执行方式 |
|------|---------|
| 常规任务 | 直接执行，完成后汇报 |
| 已授权任务 | 按授权范围执行 |
| 紧急修复 | 先执行，后汇报 |

---

## 💬 追问格式规范

### 标准追问格式

```
【追问】[任务编号] 简短描述

【现状】
- 当前情况

【选项】
- A: 方案 A(优缺点)
- B: 方案 B(优缺点)

【建议】
推荐 X，原因是...

【等待确认】
请回复 A/B 或补充说明
```

### 紧急追问格式 (简化)

```
【紧急追问】需要确认 XX

选项：A(快) / B(稳)
建议：A
原因：...

请回复 A/B
```

## ⏰ 追问时机

### 立即追问
- 任务阻塞，无法继续
- 风险操作，需要确认
- 多方案，需要决策

### 延后追问
- 非关键细节，可先执行
- 批量问题，汇总后一次性追问
- 低优先级，等待合适时机

## 🎯 追问质量评估

### 好追问 ✅
- 问题清晰，一句话能说清
- 提供选项，而非开放问题
- 给出建议，而非只抛问题
- 时机合适，不打断关键工作

### 差追问 ❌
- 问题模糊，让人困惑
- 只问问题，不给选项
- 不给建议，让人决策
- 时机不当，打断工作流

---

## 🔄 追问闭环

```
发现问题 → 判断是否追问 → 组织追问内容 → 等待回复 → 执行确认方案 → 记录归档
```

## 📝 示例

### 示例 1: 多方案选择

```
【追问】TASK-033 CAD 服务部署方案

【现状】
- LibreCAD 已安装
- 需要选择部署方式

【选项】
- A: systemd 服务 (自动启动，推荐)
- B: 手动启动 (灵活，但需每次手动)

【建议】
推荐 A，原因是：
1. 自动启动，无需手动干预
2. 崩溃后自动重启
3. 便于监控和管理

【等待确认】
请回复 A/B
```

### 示例 2: 信息缺失

```
【追问】TASK-050 知几首笔下注

【现状】
- 策略已就绪
- Polymarket API 已配置
- USDC 余额：$39.88

【缺失信息】
- 下注金额：？
- 下注市场：？

【建议】
建议首笔下注$10-20，选择流动性>$100K 的市场

【等待确认】
请回复：下注金额 + 市场类型 (或"你决定")
```
