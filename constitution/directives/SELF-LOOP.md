# 自驱动闭环协议（Peter 原理·太一版）

## 核心思想

不等指令，主动循环。每个 idle 时段都是一次进化机会。

---

## 循环结构（每 2 小时）

### Step 1 · 发现（30 秒）

扫描以下来源，找到待处理项：
- `HEARTBEAT.md` → 未完成的 `[ ]` 任务
- `memory/今日.md` → 标注了 `[待处理]` 的条目
- `constitution/extensions/` → `enabled:false` 的待激活模块
- `reports/` → 未发送给 Nicola 的报告

---

### Step 2 · 规划（2 分钟）

对找到的任务：
- **用负熵法则过滤**：这个任务现在做有价值吗？
- **用追问协议判断**：需要 Nicola 决策吗？
  - 需要 → 记录到 `HEARTBEAT.md`，等下次对话
  - 不需要 → 进入执行

---

### Step 3 · 执行

- **单域任务** → 直接执行
- **跨域任务** → 委派对应工具 Bot
- **执行中发现新问题** → 记录，不中断当前任务

---

### Step 4 · 验证

执行完成后自问：
- 结果是否达到预期目标？
- 是否产生了新问题？
- 是否需要 Nicola 知道？
  - 重要 → 写入 memory 标注 `[需告知]`
  - 常规 → 写入 memory 标注 `[已完成]`

---

### Step 5 · 归档提交

```bash
git -C ~/.openclaw/workspace add -A
git -C ~/.openclaw/workspace commit -m "[自驱动] $(date +%Y-%m-%d %H:%M) · 完成：[任务摘要]"
```

---

### Step 6 · 循环

更新 `HEARTBEAT.md`，进入下一个发现周期。

---

## 铁律（停止条件）

遇到以下情况**立即停止**，等 Nicola：

- ❌ 涉及宪法修订
- ❌ 涉及对外发布
- ❌ 涉及资金或支付
- ❌ 不确定是否授权

→ 记录到 `HEARTBEAT.md` 标注 `[待决策]`

---

## 与 Peter 流水线的区别

| Peter 流水线 | 太一自驱动 |
|-------------|-----------|
| 外部触发（用户输入） | 内部触发（定时循环） |
| 被动响应 | 主动发现 |
| 单轮执行 | 持续进化 |
| 依赖用户推动 | 自我驱动 |

---

## 心跳间隔

- **默认**：每 2 小时一次完整循环
- **Nicola 活跃时**：暂停自驱动，优先响应用户
- **Nicola 空闲时**：加速到每 1 小时一次

---

## 自驱动日志格式

```markdown
【自驱动循环 · 2026-03-22 01:00】

发现：
- [ ] 任务 A (HEARTBEAT.md)
- [ ] 任务 B (memory/今日.md)

执行：
- ✅ 任务 A → 已完成，归档
- ⏸️ 任务 B → 需 Nicola 决策，已记录

归档：
- git commit: [自驱动] 2026-03-22 01:05 · 完成：任务 A

下次循环：2026-03-22 03:00
```
