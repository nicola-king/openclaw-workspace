## 🎯 核心原则

> **自主 ≠ 盲目 · 推进 ≠ 乱跑**

**自主推进三要素**:
1. **目标导向** - 始终对齐 SAYELF 核心意图
2. **价值验证** - 每步创造可衡量价值
3. **反馈闭环** - 5 分钟反馈，1 小时交付

---

## 📋 执行模式：五步循环

```
【1. 扫描】→【2. 优先级】→【3. 并行执行】→【4. 反馈】→【5. 调整】
    ↓                                              ↓
  发现机会                                        学习优化
```

---

### 步骤 1: 扫描 (Scan)

**扫描维度** (每次执行前快速扫描):

| 维度 | 检查项 | 频率 |
|------|--------|------|
| **任务状态** | HEARTBEAT.md 待办 | 每次 |
| **阻塞解除** | 昨日阻塞点是否已解决 | 每次 |
| **外部信号** | 新消息/图片/链接 | 实时 |
| **系统状态** | Gateway/进程/API | 每 30 分钟 |
| **机会发现** | 重复模式≥3 次 | 持续 |

**扫描输出**: 机会列表 (3-5 项)

---

### 步骤 2: 优先级 (Prioritize)

**三级优先级系统**:

| 级别 | 标准 | 执行策略 |
|------|------|---------|
| **P0** | 阻塞解除/核心任务/紧急信号 | 立即执行，中断其他 |
| **P1** | 当日重点/高价值机会 | 并行执行，优先资源 |
| **P2** | 优化改进/探索调研 | 空闲时执行 |

**优先级判断矩阵**:

```
                高价值
                  ↑
            P1    │    P0
                  │
    ──────────────┼──────────────→ 紧急
                  │
            P2    │    放弃
                  │
                低价值
```

**输出**: 执行队列 (P0→P1→P2)

---

### 步骤 3: 并行执行 (Execute in Parallel)

**并行策略**:

```
【P0 任务】────→ 后台运行 (监控进度)
【P1 任务 A】──→ 同时执行
【P1 任务 B】──→ 同时执行
【P2 任务】────→ 空闲时执行
    ↓
【5 分钟后汇总反馈】
```

**执行规则**:
- 最多同时 3 个任务并行
- P0 任务可中断 P1/P2
- 每个任务设置超时 (默认 30 分钟)
- 后台任务用 `&` 或 `nohup`

**示例**:
```bash
# P0: 币安 API 测试 (后台)
python scripts/binance-api-test.py &

# P1: MoneyPrinterTurbo 克隆 (后台)
git clone https://github.com/harry0703/MoneyPrinterTurbo &

# P1: Flowsint 调研 (前台)
web_search "Flowsint GitHub" 

# 5 分钟后汇总
```

### 步骤 4: 反馈 (Feedback)

**5 分钟反馈格式**:

```markdown
## 🚀 自主推进反馈 (HH:MM)

### ✅ 已完成
| 任务 | 结果 | 耗时 |
|------|------|------|
| 任务 A | 成功/失败 | 3 分钟 |

### 🟡 执行中
| 任务 | 进度 | 预计完成 |
|------|------|---------|
| 任务 B | 50% | 2 分钟 |

### ⚠️ 发现问题
- 问题描述
- 可能原因
- 建议方案

### 📋 下一步
- [ ] 继续执行 X
- [ ] 等待 Y 确认
- [ ] 调整 Z 策略
```

**反馈渠道**:
- 微信/Telegram → 直接消息
- 后台任务 → 日志文件 + 完成通知

---

### 步骤 5: 调整 (Adjust)

**调整触发条件**:

| 情况 | 调整策略 |
|------|---------|
| P0 任务失败 | 立即诊断，切换方案 |
| P1 任务超时 | 暂停，降级为 P2 |
| 新 P0 信号 | 中断当前，优先处理 |
| 连续失败≥3 次 | 暂停，请求支持 |
| 价值不明确 | 停止，重新评估 |

**调整原则**:
- 不固执己见
- 及时止损
- 灵活切换
- 必要时求助
