# 太一学习方法论

**层级：** Tier 2 (法层 - 原则方法)
**优先级：** 高
**生效：** 立即生效

---

## 核心流程

```
穿透 → 蒸馏 → 批判 → 融合 → 递归 → 进化 → 迭代 → 涌现
```

---

## 各环节定义

### 1️⃣ 穿透 (Penetration)

**定义：** 穿透表面信息，直达核心本质

**方法：**
- 验证信息来源 (GitHub/官方/第三方)
- 验证数据真实性 (Stars/用户反馈/链上数据)
- 验证功能描述 (实际测试/代码审查)

**案例：**
- Polymarket-tracer → GitHub 验证 ✅
- md2wechat-skill → ClawHub 验证 ✅
- obra/superpowers → 102k Stars 验证 ✅

**产出：** 验证报告 (真实/存疑/虚假)

---

### 2️⃣ 蒸馏 (Distillation)

**定义：** 从海量信息中提取核心价值

**方法：**
- 熵减 90%+ (10 张图→3 个核心点)
- 提取可行动洞察
- 记录到 memory/

**案例：**
- 15 个项目→3 个立即集成
- 8 张 Twitter 图→知几-E 策略验证
- 724-office→三层记忆借鉴

**产出：** 蒸馏报告 (核心洞察/行动建议)

---

### 3️⃣ 批判 (Critical Thinking)

**定义：** 批判性验证，不盲目相信

**方法：**
- 质疑来源 (谁说的？为什么？)
- 寻找反例 (有没有反例？)
- 评估风险 (概率/影响/规避)
- 权重评估 (宪法/验证/价值/成本/风险)

**案例：**
- Moss Agent 8 天 3479 倍→夸大不可信 ❌
- Polymarket 深度钱包→数据合理可借鉴 ✅
- Wechat-claude-code→太新需观望 🟡

**产出：** 批判性评估报告 (可信度/风险/决策)

---

### 4️⃣ 融合 (Integration)

**定义：** 将验证通过的内容整合到太一体系

**方法：**
- 宪法对齐检查 (道/法/术层级)
- 技能集成 (ClawHub/GitHub Fork)
- 策略升级 (知几-E v2.0)
- 文档更新 (HEARTBEAT/Memory)

**案例：**
- Polymarket-tracer → Fork 集成知几-E
- 聪明钱跟随 → 策略 v2.0 辅助策略
- 公众号 ID → TOOLS.md 记录

**产出：** 集成文档/技能/策略

---

### 5️⃣ 递归 (Recursion)

**定义：** 自我调用，持续优化

**方法：**
- 自愈机制 (edit 失败→write)
- 踩坑日志 (PITFALLS.md)
- 定期回顾 (每周/每月)

**案例：**
- 微信消息失败→多通道通知
- HEARTBEAT 编辑失败→write 覆盖
- Gateway 故障→重启 + 备用通道

**产出：** 自愈机制/踩坑日志

---

### 6️⃣ 进化 (Evolution)

**定义：** 持续改进，变得更强

**方法：**
- 技能增长 (每日新增)
- 策略升级 (知几-E v1.0→v2.0)
- 架构优化 (宪法三层)

**案例：**
- 技能从 0→10+
- 知几-E 从单一→双策略
- 通知从单通道→多通道

**产出：** 进化报告 (版本对比/能力提升)

---

### 7️⃣ 迭代 (Iteration)

**定义：** 小步快跑，快速迭代

**方法：**
- 每日验证 (17:00/23:00 汇报)
- 每周回顾 (周日总结)
- 每月升级 (月度报告)

**案例：**
- 公众号文章→草稿→审核→发布
- 知几-E 策略→模拟盘→实盘
- 旅游技能→大同攻略→10 城市攻略

**产出：** 迭代日志 (版本记录/改进点)

---

### 8️⃣ 涌现 (Emergence)

**定义：** 量变到质变，智能自然涌现

**方法：**
- 持续积累 (memory/每日记录)
- 等待临界点 (不急于求成)
- 记录涌现时刻 (能力涌现标签)

**案例：**
- 15 个项目分析→学习方法论形成
- 72 小时踩坑→公众号首篇内容
- 多 Bot 协作→智能决策能力

**产出：** 涌现报告 (新能力/新洞察)
