## 🛡️ 6 大核心保障机制

### [机制 1: 5 分钟进度汇报] ✅ 强制

**要求**:
- 每 5 分钟自动发送进度汇报
- 包含：当前任务、进度条、状态、下一步、预计完成
- 通道：Telegram (主)、微信 (可选)、飞书 (可选)

**Cron 配置**:
```bash
openclaw cron add --name "auto-exec-5m" \
  --every "5m" \
  --agent "taiyi" \
  --message "python3 scripts/auto-exec-report.py"
```

**状态文件**: `/tmp/auto-exec-status.json`

**验收**:
- [ ] 每 5 分钟自动发送
- [ ] 进度条可视化
- [ ] 当前任务清晰
- [ ] 下一步明确

---

### [机制 2: 任务自动发现] ✅ 强制

**任务来源**:
1. `HEARTBEAT.md` - P0 核心任务
2. `memory/residual.md` - P1/P2 任务
3. `memory/core.md` - 长期任务
4. 用户指令 - 临时任务

**发现频率**: 每 5 分钟检查一次

**任务状态机**:
```
pending → running → completed
              ↓
           blocked (需人工介入)
```

**验收**:
- [ ] 新任务 5 分钟内发现
- [ ] 优先级正确排序 (P0 > P1 > P2)
- [ ] 阻塞任务自动标记

---

### [机制 3: 自动执行触发] ✅ 强制

**触发条件**:
- 新任务出现 (状态 pending)
- 阻塞解除 (状态 blocked → pending)
- 定时任务到达
- 用户指令

**执行流程**:
```
1. 读取任务列表
2. 按优先级排序 (P0 > P1 > P2)
3. 选择最高优先级 pending 任务
4. 执行任务
5. 更新状态
6. 5 分钟后汇报
```

**原则**:
- ✅ 不等待确认
- ✅ 不事前请示
- ✅ 执行后汇报
- ✅ 阻塞自动跳过

**验收**:
- [ ] 发现任务立即执行
- [ ] 0 等待确认
- [ ] 阻塞任务自动跳过

## 📋 执行原则

### [自主性原则] ✅ 宪法级

```
1. 不等待确认 - 发现任务立即执行
2. 不事前请示 - 执行后汇报结果
3. 不表演过程 - 只输出有价值内容
4. 不重复劳动 - 已完成任务不重做
```

**违反后果**: 违反负熵法则

---

### [优先级原则] ✅ 宪法级

```
1. P0 优先 - 核心任务先执行
2. 阻塞跳过 - 阻塞任务自动跳过
3. 并行执行 - 可同时执行多任务
4. 及时归档 - 完成立即归档
```

**违反后果**: 任务延误

---

### [透明性原则] ✅ 宪法级

```
1. 进度可见 - 每 5 分钟更新状态
2. 问题透明 - 阻塞立即上报
3. 结果可追溯 - 归档到 memory
4. 数据可验证 - 状态文件公开
```

**违反后果**: 信任损失
