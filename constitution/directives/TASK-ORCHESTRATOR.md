# 任务编排法则 (Task Orchestrator Directive)

> **Tier**: 1 (永久核心) | **版本**: 1.0.0 | **创建时间**: 2026-04-03 | **太一 AGI v5.0**

---

## 🎯 核心原则

**任务编排 > 手动分配 · 流程化 > 临时性 · 可追踪 > 黑盒执行**

Task Orchestrator 是太一 AGI 的**智能任务编排引擎**，确保所有任务从分解到验收的完整闭环。

---

## 🔄 六阶段流程

### 阶段 1: 任务分解 (Decomposition)

**目标**: 将模糊的自然语言任务拆解为可执行的子任务

**标准**:
- 每个子任务有明确 ID
- 任务类型清晰 (技术/内容/数据/交易)
- 交付物定义明确
- 工作量估算合理

**触发词**:
- "分解任务..."
- "规划..."
- "安排..."

---

### 阶段 2: 智能分配 (Allocation)

**目标**: 根据 Bot 职责域自动匹配最合适的执行者

**Bot 职责映射**:

| Bot | 职责域 | Emoji |
|-----|--------|-------|
| 素问 | 技术开发 | 💻 |
| 山木 | 内容创作 | 🎨 |
| 罔两 | 数据采集 | 📊 |
| 知几 | 量化交易 | 📈 |
| 羿 | 监控追踪 | 🏹 |
| 守藏吏 | 资源配置 | 📚 |
| 庖丁 | 成本核算 | 💰 |
| 太一 | 统筹调度 | 🧠 |

**分配规则**:
1. 单一职责：每个任务有且仅有一个主责 Bot
2. 协作机制：跨域任务指定协作 Bot
3. 负载均衡：避免单 Bot 过载 (>5 任务)

---

### 阶段 3: 分发送达 (Dispatch)

**目标**: 创建标准化任务委派单并送达 Bot inbox

**委派单内容**:
- 任务 ID 和描述
- 优先级和截止时间
- 交付物清单
- 汇报要求
- 纠偏机制说明

**送达确认**:
- Bot 需在 5 分钟内确认接收
- 超时 → 自动提醒
- 10 分钟未确认 → 太一介入

---

### 阶段 4: 执行追踪 (Tracking)

**目标**: 实时监控任务执行进度

**追踪频率**: 每 30 分钟自动检查

**追踪内容**:
- 扫描 Bot outbox 汇报
- 更新任务状态
- 检测进度滞后
- 生成追踪报告

**状态定义**:
- ⏳ 待开始 (pending)
- 🟡 执行中 (in_progress)
- ✅ 已完成 (completed)
- 🔴 已阻塞 (blocked)

---

### 阶段 5: 成果验收 (Validation)

**目标**: 验证 Bot 交付成果是否符合标准

**验收维度**:
1. **完整性** (Completeness): 所有交付物齐全
2. **时效性** (Timeliness): 在截止时间前完成
3. **文档化** (Documentation): 报告/说明完整
4. **质量** (Quality): 符合预定义标准

**验收结果**:
- ✅ 通过 (≥80%)
- 🟡 需改进 (50-80%)
- 🔴 失败 (<50%)

---

### 阶段 6: 汇报归档 (Reporting)

**目标**: 生成报告并归档到知识库

**归档位置**:
- `reports/task-{id}-report.md` - 任务报告
- `memory/{bot}/task-{id}.md` - Bot 记忆
- `HEARTBEAT.md` - 待办更新

**汇报内容**:
- 任务概述
- 执行过程
- 产出统计
- 经验总结

---

## 🚨 纠偏机制

### 异常检测

| 异常类型 | 触发条件 | 严重级别 |
|---------|---------|---------|
| 未确认 | 分发后 10 分钟未确认 | P1 |
| 进度滞后 | 落后计划 30 分钟 | P2 |
| 进度滞后 | 落后计划 1 小时 | P1 |
| 进度滞后 | 落后计划 2 小时 | P0 |
| 阻塞 | Bot 上报阻塞 | P0 |
| 超时 | 超过截止时间 | P1 |

### 处理策略

| 级别 | 响应时间 | 处理人 | 动作 |
|------|---------|-------|------|
| P0 | 立即 | 太一 | 立即介入 |
| P1 | 15 分钟 | 太一 | 协调资源 |
| P2 | 30 分钟 | 自动 | 自动提醒 |

### 纠偏流程

```
检测异常 → 分级告警 → 介入处理 → 记录原因 → 优化策略
```

## 📊 监控指标

| 指标 | 目标值 | 计算方式 |
|------|-------|---------|
| 任务分解准确率 | ≥95% | 正确分解数/总任务数 |
| 分配合理率 | ≥90% | Bot 反馈合理数/总分配数 |
| 送达确认率 | 100% | 确认接收数/总分发数 |
| 执行完成率 | ≥85% | 按时完成数/总任务数 |
| 验收通过率 | ≥90% | 一次通过数/总验收数 |
| 纠偏及时率 | ≥95% | 及时处理数/总异常数 |

## 🛠️ 工具集成

### Task Orchestrator Skills

**位置**: `skills/task-orchestrator/`

**核心组件**:
- `orchestrator.py` - 编排引擎
- `task-tracker.py` - 执行追踪
- `validator.py` - 成果验收
- `correction.py` - 纠偏机制

### CLI 命令

```bash
# 任务分解
python3 orchestrator.py decompose "<任务描述>"

# 任务分发
python3 orchestrator.py dispatch <bot>

# 进度追踪
python3 task-tracker.py scan

# 成果验收
python3 validator.py validate <bot> <task_id>

# 纠偏检测
python3 correction.py detect

# 生成报告
python3 orchestrator.py summary
```

### Cron 定时任务

**频率**: 每 30 分钟

**脚本**: `skills/task-orchestrator/scripts/orchestrator-cron.sh`

**执行内容**:
1. 扫描 Bot 任务状态
2. 检测异常
3. 处理异常
4. 生成追踪/纠偏/汇总报告
5. 更新 HEARTBEAT.md
