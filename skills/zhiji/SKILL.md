## ⚔️ 铁律

```
NO TRADING CODE WITHOUT A FAILING TEST FIRST
```

**写交易代码前没测试？删除。重来。**

**无例外：**
- 不因为"急"而跳过测试
- 不因为"老板说"而跳过测试
- 不因为"复杂"而跳过测试
- 不接受"手动测过了"
- 不接受"测试后补"
- 不因为"已经写了一半"而跳过测试

**违反 = 删除代码，重来。**

## 🚩 红旗列表 - STOP and Start Over

**出现以下任何一条，立即停止，请示太一：**

- [ ] "这个很急，先实现再补测试"
- [ ] "我老板/客户说不用测试"
- [ ] "逻辑太复杂，测试写不出来"
- [ ] "我已经手动测过了"
- [ ] "测试只是形式，精神更重要"
- [ ] "这次情况特殊"

**所有这些都意味着：你在 rationalize，不是在执行。**

---

## 📋 合理化表格

| 借口 | 现实 |
|------|------|
| "很急，先实现" | 越急越要测试，错了更耽误时间 |
| "老板说不用测" | 老板不为你的错误买单 |
| "太复杂测不了" | 越复杂越要测试，否则死得惨 |
| "手动测过了" | 手动测试不可重复，不是测试 |
| "测试后补" | 事后测试 = "这代码干嘛？" vs "应该干嘛？" |
| "已经写了一半" | 沉没成本，删除重写更快 |
| "这次情况特殊" | 每次都觉得特殊，所以每次都错 |
