## 🎯 一、核心价值提取

### 1.1 Agent-Reach 是什么

**一句话定义：**
> 给 AI Agent 装上互联网眼睛的脚手架（scaffolding）

**核心定位：**
- 不是框架，是脚手架
- 不是包装层，是直接调用上游工具
- 每个渠道可插拔替换

### 1.2 核心能力蒸馏

| 能力 | 描述 | 实现方式 |
|------|------|---------|
| **读网页** | 阅读任意网页 | Jina Reader |
| **读 YouTube** | 字幕提取 + 视频搜索 | yt-dlp |
| **读 Twitter** | 读推文/搜索/发推 | xreach (Cookie 登录) |
| **读 GitHub** | 仓库/Issue/PR | gh CLI |
| **读 Reddit** | 搜索/读帖子 | Exa + JSON API |
| **读小红书** | 阅读/搜索/发帖/评论/点赞 | xiaohongshu-mcp |
| **读抖音** | 视频解析/无水印下载 | douyin-mcp-server |
| **读微信公众号** | 搜索 + 阅读全文 | wechat-article-for-ai + miku_ai |
| **读微博** | 热搜/搜索/评论 | 爬虫 |
| **读 RSS** | 阅读任意 RSS/Atom | feedparser |
| **全网搜索** | AI 语义搜索 | Exa via mcporter |

### 1.4 设计理念蒸馏

**核心思想：**
> 帮你把选型和配置的活儿做完了

**关键原则：**
1. **可插拔：** 每个渠道独立文件，不满意换掉
2. **直接调用：** 不经过包装层，直接调用上游工具
3. **免费优先：** 所有 API 免费，唯一可能花钱是服务器代理 ($1/月)
4. **本地优先：** Cookie 本地存储，不上传不外传
5. **自诊断：** `agent-reach doctor` 一条命令诊断状态

## 📌 三、蒸馏结论

### 3.1 Agent-Reach 的核心价值

**一句话：**
> 把 AI Agent 访问互联网的门槛降到最低

**三个关键词：**
1. **脚手架** - 不是框架，是工具集合
2. **可插拔** - 每个渠道独立，可替换
3. **免费** - 优先免费方案，降低成本

### 3.2 毕方的定位差异

| 维度 | Agent-Reach | 毕方 |
|------|-------------|------|
| **定位** | 通用互联网访问 | 小红书爆款 + 复刻 |
| **目标** | 让 Agent 能访问 | 让内容能变现 |
| **核心** | 渠道覆盖 | 爆款公式 |
| **价值** | 降低访问门槛 | 创造盈利价值 |

### 3.3 复用策略

**混合复用（推荐）：**
```
毕方 v0.1 = 
  30% 复用 (xiaohongshu-mcp + yt-dlp + Jina Reader) +
  70% 自研 (爆款发现 + 内容分析 + 视频复刻)
```
