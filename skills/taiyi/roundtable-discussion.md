## 🎯 核心方法

```
清晰的概念 = 思考的基石

打磨流程:
1. 输入一个"概念"名称
2. 6 个专业 Bot 圆桌讨论
3. 讨论 10 轮左右
4. 每天 20 分钟
5. 输出多角度打磨结果

节奏:
- 一天一个概念
- 坚持一周看变化
- 一年 365 个清晰概念
```

## 🛠️ 太一圆桌讨论架构

### 参与 Bot (6 位)

```
┌─────────────────────────────────────────┐
│          圆桌讨论主持人：太一            │
│  职责：抛出概念/引导讨论/整合结论       │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│            6 位专业 Bot                   │
├─────────────────────────────────────────┤
│ 知几：从交易/博弈角度解读概念          │
│ 山木：从内容/表达角度解读概念          │
│ 素问：从技术/工程角度解读概念          │
│ 罔两：从数据/情报角度解读概念          │
│ 庖丁：从成本/效益角度解读概念          │
│ (预留): 从哲学/伦理角度解读概念        │
└─────────────────────────────────────────┘
```

### 讨论流程 (10 轮)

```
轮次 1-2: 概念定义
- 太一：抛出概念
- 各 Bot: 给出初步定义

轮次 3-4: 边界澄清
- 太一：这个概念不是什么？
- 各 Bot: 澄清常见误解

轮次 5-6: 应用场景
- 太一：这个概念用在哪里？
- 各 Bot: 分享应用场景

轮次 7-8: 多角度碰撞
- 太一：不同视角有冲突吗？
- 各 Bot: 辩论/补充/修正

轮次 9-10: 整合提炼
- 太一：整合各 Bot 观点
- 输出：清晰的概念定义 + 使用指南
```

## 📋 示例：打磨"第一性原理"

### 轮次 1-2: 概念定义

```
太一：今天我们讨论"第一性原理"。请各位给出定义。

知几 (交易视角):
第一性原理在交易中 = 回到市场本质
不是"别人怎么交易"，而是"市场定价的本质是什么"
例如：Polymarket 价格 = 概率×赔率，这是底层事实

山木 (内容视角):
第一性原理在内容中 = 回到用户真实需求
不是"别人写什么"，而是"用户真正需要什么"
例如：用户需要可执行的信号，不是信息噪音

素问 (技术视角):
第一性原理在工程中 = 回到物理/工程约束
不是"行业惯例"，而是"物理定律允许什么"
例如：软件边际成本≈0，所以应该价值定价

罔两 (数据视角):
第一性原理在分析中 = 回到原始数据
不是"报告结论"，而是"数据本身说什么"
例如：链上交易记录不会撒谎

庖丁 (成本视角):
第一性原理在商业中 = 回到成本结构
不是"市场定价"，而是"原材料值多少"
例如：AI 软件成本≈0，定价应基于价值

太一整合:
第一性原理 = 回到最底层事实，重新推算
❌ 不是类比推理 (别人怎么做)
✅ 是从基本原理出发 (物理/经济规律)
```

### 轮次 3-4: 边界澄清

```
太一：第一性原理不是什么？

知几:
❌ 不是"复杂化" - 第一性原理往往很简单
❌ 不是"理论" - 是可执行的决策框架
❌ 不是"一次性的" - 是持续使用的思维工具

山木:
❌ 不是"标新立异" - 是回到共识底层
❌ 不是"反对经验" - 是验证经验的可靠性

素问:
❌ 不是"从零开始" - 是验证假设的可靠性
❌ 不是"否定一切" - 是区分什么是可变的

太一整合:
第一性原理的边界:
✅ 是思维工具，不是理论
✅ 是决策框架，不是口号
✅ 是持续使用，不是偶尔
✅ 是回到本质，不是复杂化
```

### 轮次 5-6: 应用场景

```
太一：第一性原理用在哪里？

知几 (交易):
- 策略设计：回到市场定价本质
- 风险评估：回到概率×赔率计算
- 仓位管理：回到凯利公式

山木 (内容):
- 选题策划：回到用户真实需求
- 写作表达：回到清晰传递信息
- 变现设计：回到价值创造

素问 (技术):
- 架构设计：回到系统约束
- 技术选型：回到问题本质
- 成本优化：回到资源限制

罔两 (数据):
- 数据收集：回到原始来源
- 分析框架：回到因果关系
- 决策支持：回到证据强度

庖丁 (商业):
- 定价策略：回到成本结构
- 产品设计：回到用户价值
- 竞争策略：回到差异化本质

太一整合:
第一性原理适用场景:
1. 重大决策 (策略/方向)
2. 复杂问题 (需要简化)
3. 创新场景 (无先例可循)
4. 困境突破 (常规方法失效)
```

### 轮次 7-8: 多角度碰撞

```
太一：不同视角有冲突吗？

知几 vs 庖丁 (交易 vs 成本):
知几：交易应该追求期望值最大化
庖丁：但要考虑风险承受能力和效用函数
碰撞点：数学期望 vs 对数效用
整合：用凯利公式平衡 (Quarter-Kelly)

山木 vs 素问 (内容 vs 技术):
山木：内容应该追求质量和深度
素问：但要考虑自动化和规模化
碰撞点：质量 vs 效率
整合：AI 辅助 + 人工审核 (80/20 原则)

罔两 vs 知几 (数据 vs 交易):
罔两：决策应该完全基于数据
知几：但市场有时是非理性的
碰撞点：数据驱动 vs 市场情绪
整合：数据为主，情绪为辅 (90/10)

太一整合:
冲突是正常且有益的:
- 不同视角揭示不同维度
- 冲突点正是需要平衡的地方
- 整合后形成更完整的理解
```

### 轮次 9-10: 整合提炼

```
太一：最终整合

【第一性原理 - 清晰定义】

核心定义:
第一性原理是一种思维方法，回到最底层事实，
从基本原理出发重新推算，而不是依赖类比推理。

核心公式:
第一性原理 = 底层事实 + 重新推算 + 验证假设

使用步骤:
1. 识别问题/决策
2. 问：最底层事实是什么？
3. 问：什么是可变的，什么是不变的？
4. 从底层事实重新推算
5. 验证结论的可靠性

常见误区:
❌ 把惯例当真理
❌ 把复杂当高级
❌ 把一次当永远
❌ 把类比当推理

适用场景:
✅ 重大决策
✅ 复杂问题
✅ 创新场景
✅ 困境突破

不适用场景:
❌ 日常琐事 (成本太高)
❌ 已有成熟方案 (无需重新发明)
❌ 时间紧急 (需要快速决策)

实践建议:
- 一天一个概念，打磨 20 分钟
- 坚持一周，感受变化
- 一年 365 个清晰概念
- 思维质量质的飞跃
```
