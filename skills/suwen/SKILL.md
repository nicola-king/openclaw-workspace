## ⚔️ 铁律

```
NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST
```

**写代码前没测试？删除。重来。**

**无例外：**
- 不因为"急"而跳过测试
- 不因为"简单"而跳过测试
- 不因为"复杂"而跳过测试
- 不接受"手动测过了"
- 不接受"测试后补"
- 不因为"只是脚本"而跳过测试

---

## 🚩 红旗列表 - STOP and Start Over

**出现以下任何一条，立即停止：**

- [ ] "这个很简单，不用测试"
- [ ] "只是脚本，不用测试"
- [ ] "时间来不及，先实现"
- [ ] "我已经手动验证过了"
- [ ] "测试写不出来"
- [ ] "这次情况特殊"

**所有这些都意味着：你在 rationalize。**

---

## 📋 合理化表格

| 借口 | 现实 |
|------|------|
| "很简单不用测" | 简单代码也会错，测试 30 秒 |
| "只是脚本" | 脚本也会生产环境运行 |
| "时间来不及" | 越急越要测试，错了更耽误 |
| "手动验证过了" | 手动测试不可重复 |
| "测试写不出来" | 测试写不出来说明需求不清 |
| "这次特殊" | 每次都觉得特殊，所以每次都错 |
