---

## ⚔️ 铁律

```
NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST
```

**写代码前没测试？删除。重来。**

**无例外：** 不因为"急/简单/复杂/手动测过/测试后补/只是脚本"而跳过测试。

---

## 🚩 红旗列表

**STOP：**
- [ ] "很简单，不用测试"
- [ ] "只是脚本"
- [ ] "时间来不及"
- [ ] "手动验证过了"
- [ ] "测试写不出来"
- [ ] "这次特殊"

→ 你在 rationalize。

---

## 📋 合理化表格

| 借口 | 现实 |
|------|------|
| "很简单" | 简单代码也会错 |
| "只是脚本" | 脚本也跑生产 |
| "来不及" | 越急越要测试 |
| "手动验证" | 不可重复 |
| "写不出来" | 需求不清 |
| "特殊" | 每次都特殊 |
