## 🧠 TurboQuant v2 压缩策略

### 核心优化技术

1. **城市编码**: 字符串→单字符 ID (A-Z)
   - 例：`beijing` → `A`, `shanghai` → `B`
   - 节省：~80% 城市名字符

2. **日期压缩**: YYYY-MM-DD→相对天数
   - 例：`2026-03-25` → `2` (基准日期 +2 天)
   - 节省：~70% 日期字符

3. **数值量化**: 浮点数→整数 (×10)
   - 例：`23.5` → `235`
   - 去除小数点，减少精度冗余

4. **差值编码**: 存储相对变化
   - 每个城市提取基准值
   - 只存储与基准的差值
   - 差值通常较小，压缩效率更高

5. **结构优化**: 对象→数组
   - 去除字段名冗余
   - 例：`{"c":"A","d":2}` → `["A",2,...]`

6. **zlib 压缩**: 最终二进制压缩
   - Level 9 最大压缩
   - 利用数据重复模式

### 两层架构设计

```
┌─────────────────────────────────────┐
│  主成分层 (m + c)                    │
│  • metadata: 城市映射 + 基准值       │
│  • core: 紧凑数组 [城市，日期，差值]  │
│  • 可独立使用，快速决策              │
├─────────────────────────────────────┤
│  残差层 (r)                          │
│  • weather_codes: 天气代码数组       │
│  • 按需加载，精确修复                │
└─────────────────────────────────────┘
```
