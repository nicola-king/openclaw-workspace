## 🧮 384 阶段计算

```
64 状态 × 6 阶段 = 384 阶段

每个状态的 6 个阶段：
- Step 1: 刚开始不对劲
- Step 2: 逐渐察觉问题
- Step 3: 开始怀疑路径
- Step 4: 尝试调整方式
- Step 5: 逐步适应
- Step 6: 进入新状态

阶段索引：1-6（对应传统概念的"初爻"到"上爻"）
```

### 阶段推进逻辑

```python
def calculate_step(answers: List[Dict]) -> int:
    """
    根据测试答案计算当前阶段（1-6）
    
    底层逻辑：
    - 初爻（Step 1）：初始状态
    - 二爻（Step 2）：逐渐显现
    - 三爻（Step 3）：开始怀疑
    - 四爻（Step 4）：尝试调整
    - 五爻（Step 5）：逐步适应
    - 上爻（Step 6）：完成转化
    """
    
    # 基于答案的情绪强度/问题严重性计算
    score = sum(answer['intensity'] for answer in answers)
    
    if score < 10:
        return 1  # 初爻
    elif score < 20:
        return 2  # 二爻
    elif score < 30:
        return 3  # 三爻
    elif score < 40:
        return 4  # 四爻
    elif score < 50:
        return 5  # 五爻
    else:
        return 6  # 上爻
```

### 匹配算法

```python
def match_stage(answers: List[Dict]) -> Dict:
    """
    情景匹配算法（替代传统"起卦"）
    
    输入：用户答案列表
    输出：匹配的情景 + 阶段 + 置信度
    """
    
    # Step 1: 计算 4 大类型得分
    type_scores = {
        '调整型': 0,
        '过渡型': 0,
        '观察型': 0,
        '决策型': 0
    }
    
    for answer in answers:
        type_scores[answer['type']] += answer['score']
    
    # Step 2: 确定主类型
    main_type = max(type_scores, key=type_scores.get)
    
    # Step 3: 在该类型中选择具体情景（64 状态之一）
    stage_index = calculate_stage_index(answers)
    stage = stages[main_type][stage_index]
    
    # Step 4: 计算阶段（1-6，对应 6 爻）
    step = calculate_step(answers)
    
    # Step 5: 计算置信度
    confidence = calculate_confidence(answers)
    
    return {
        'stage_id': stage['id'],
        'stage_name': stage['name'],
        'type': main_type,
        'step': step,  # 1-6
        'confidence': confidence,
        'next_stages': get_next_stages(stage['id']),  # 情景转换
        'contrast_stages': get_contrast_stages(stage['id']),  # 对比情景
    }
```
