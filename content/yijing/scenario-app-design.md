## 🔐 底层逻辑映射（合规化设计）

**重要**：以下为系统底层逻辑，**不对用户展示**，仅内部使用。

┌──────────────────┬──────────────┬─────────────────┬──────────┐
│ 合规用语         │ 底层逻辑     │ 传统概念        │ 状态     │
├──────────────────┼──────────────┼─────────────────┼──────────┤
│ 今日情景         │ 状态系统     │ 易经            │ 核心     │
│ 64 状态          │ 状态分类     │ 64 卦           │ 核心     │
│ 384 阶段         │ 状态×阶段    │ 384 爻          │ 核心     │
│ 情景测试         │ 匹配算法     │ 起卦            │ 功能     │
│ 情景转换         │ 状态迁移     │ 变卦            │ 功能     │
│ 对比情景         │ 相似/相反    │ 错卦/综卦       │ 功能     │
└──────────────────┴──────────────┴─────────────────┴──────────┘

**合规原则**:
```
❌ 禁止出现：易经、卦、爻、占卜、算命、迷信
✅ 使用用语：情景、状态、阶段、测试、匹配、心理学
```

**设计逻辑**:
```
64 状态 × 6 阶段 = 384 阶段

每个状态代表一种人生情景
每个阶段代表该情景内的发展步骤
用户通过测试匹配到最符合的状态 + 阶段
```

**为什么这样设计**:
```
1. 合规性：避免迷信/占卜相关审核风险
2. 普适性：现代心理学语言更易被接受
3. 实用性：提供可执行行动建议，不只是"解读"
4. 传承性：底层逻辑保留传统智慧结构
```

## 🧪 情景测试设计 (替代"起卦")

### 测试流程

```
用户进入小程序
  ↓
选择测试类型:
  - 快速测试 (3 题，30 秒)
  - 标准测试 (12 题，2 分钟)
  - 深度测试 (36 题，5 分钟)
  ↓
答题 (选择题/评分题)
  ↓
算法匹配最符合的情景
  ↓
输出情景报告
```

### 快速测试示例 (3 题)

**Q1: 你最近的状态更接近？**
- A. 努力但没有回报
- B. 进展顺利但有波动
- C. 感到迷茫不知道方向
- D. 疲惫想停下来休息

**Q2: 你当前的主要困扰是？**
- A. 外部阻力 (人际关系/资源不足)
- B. 内部困惑 (方向/方法/信心)
- C. 时机问题 (太早/太晚/等待)
- D. 决策压力 (需要选择/害怕选错)

**Q3: 你希望得到什么帮助？**
- A. 具体行动建议
- B. 心理层面的理解
- C. 趋势和方向判断
- D. 情绪支持和鼓励

**算法匹配**:
- 主要选 A → 调整型情景
- 主要选 B → 过渡型情景
- 主要选 C → 观察型情景
- 主要选 D → 决策型情景
