## 🎯 五大核心原则

### 1. 第一性原理 ✅

**定义：** 代码设计和实践中的必要性确认

**太一实践：**
- ✅ 每行代码都有必要性
- ✅ 避免过度工程化
- ✅ 核心逻辑保持简单

**检查清单：**
```markdown
添加新功能前问：
- [ ] 这个功能是否必要？
- [ ] 是否简化了系统？
- [ ] 是否符合核心目标？
- [ ] 是否有更简单的实现？
```

---

### 2. 不向后兼容 🟡

**定义：** 有效避免屎山，Agent 重构清理

**太一实践：**
- ✅ 版本迭代（v2.1 → v2.2 → v2.3）
- 🟡 破坏性变更机制
- 🟡 迁移脚本

**实施指南：**
```python
# 版本迁移脚本
def migrate_v23_to_v30():
    """
    v3.0 破坏性变更：
    - 移除旧 API
    - 配置格式变更
    - 自动迁移旧数据
    """
    pass
```

---

### 3. 唯一事实源 ✅

**定义：** 避免日后迭代挖坑

**太一实践：**
- ✅ `.taiyi/settings.json` 唯一配置源
- ✅ `memory/2026-03-24.md` 唯一记忆源
- ✅ `HEARTBEAT.md` 唯一待办源

**优势：**
- ✅ 避免配置冲突
- ✅ 避免数据不一致
- ✅ 便于维护

---

### 4. 原子化提交 ✅

**定义：** 整个代码的 Commit 以最小单位实现

**太一实践：**
- ✅ Git 提交规范（已创建）
- ✅ 功能分支开发
- ✅ 小步快跑

**提交规范：**
```bash
# ✅ 好的提交
git commit -m "feat: 添加日损熔断"
git commit -m "fix: 修复滑点计算"
git commit -m "docs: 更新配置文档"

# ❌ 坏的提交
git commit -m "更新了很多东西"
```

**文档：** `docs/git-commit-guidelines.md`

---

### 5. 测试用例先行 🟡

**定义：** 让 AI 大声背诵三遍，会有奇效

**太一实践：**
- ✅ 测试框架创建
- ✅ 核心参数测试
- 🟡 覆盖率提升

**测试清单：**
```python

def test_confidence_threshold():
    """置信度阈值必须是 96%"""
    assert engine.confidence_threshold == 0.96
    
def test_edge_threshold():
    """优势阈值必须覆盖 2.5% 成本"""
    assert engine.edge_threshold == 0.045
    
def test_daily_loss_limit():
    """日损 10% 熔断"""
```

## 📋 开发流程

### 1. 需求分析

```markdown
- [ ] 是否符合第一性原理？
- [ ] 是否必要？
- [ ] 是否有更简单的实现？
```

### 2. 测试先行

```python
# 1. 先写测试
def test_new_feature():
    pass

# 2. AI 背诵三遍测试逻辑
# 3. 运行测试（失败）
```

### 3. 实现功能

```python
# 1. 最小实现
# 2. 运行测试（通过）
# 3. 重构优化
```

### 4. 原子提交

```bash
git add .
git commit -m "feat: 实现新功能"
```

### 5. 代码审查

```markdown
- [ ] 是否符合所有原则？
- [ ] 测试是否覆盖？
- [ ] 文档是否更新？
```

## ⚠️ 避免的陷阱

| 陷阱 | 问题 | 正确做法 |
|------|------|----------|
| **过度设计** | 违反第一性原理 | 保持简单 |
| **向后兼容** | 屎山积累 | 大胆重构 |
| **多事实源** | 数据不一致 | 唯一事实源 |
| **大提交** | 难以审查 | 原子化提交 |
| **无测试** | 质量不可控 | 测试先行 |

---

## ✅ 最佳实践

**1. 简单优于复杂**
```python
# ✅ 简单直接
return confidence >= 0.96

# ❌ 过度复杂
if confidence >= self.get_threshold_manager().get_default_threshold():
    return True
```

**2. 测试覆盖核心逻辑**
```python
# 核心参数必须测试
def test_critical_params():
    assert engine.confidence_threshold == 0.96
    assert engine.edge_threshold == 0.045
```

**3. 文档同步更新**
```markdown
- [ ] 代码完成
- [ ] 测试通过
- [ ] 文档更新
- [ ] 提交代码
```
