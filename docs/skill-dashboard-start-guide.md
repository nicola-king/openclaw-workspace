## 🎯 功能说明

现在你可以在 **太一 Skill 管理中心** 可视化界面中：

1. **点击"启动"按钮** → 真正启动 Skill 应用
2. **点击"打开"按钮** → 在浏览器中打开应用
3. **点击"日志"按钮** → 查看应用运行日志
4. **点击"状态"按钮** → 查看应用运行状态

---

## 🚀 使用步骤

### 步骤 1: 访问 Skill 管理中心

打开浏览器访问：
```
http://localhost:5002
```

### 步骤 2: 找到要启动的 Skill

- 使用搜索框快速查找
- 查看 Skill 卡片信息

### 步骤 3: 启动应用

点击 **"▶️ 启动"** 按钮

系统会自动：
- ✅ 查找可执行文件 (main.py/app.py/server.py/run.py 等)
- ✅ 检测端口配置
- ✅ 启动应用进程
- ✅ 保存进程信息

### 步骤 4: 在浏览器中打开

启动成功后，点击 **"🌐 打开"** 按钮

系统会：
- ✅ 在新标签页打开应用
- ✅ 显示应用界面

## 🔍 端口自动检测

系统会自动从代码中检测端口号：

```python
# 支持以下格式
port = 5000
PORT = 8080
app.run(port=3000)
:8000
```

**默认端口映射**：
- 包含 `5000` → 5000
- 包含 `3000` → 3000
- 包含 `8080` → 8080
- 包含 `8000` → 8000

## 🛠️ API 接口

### 启动应用
```
POST /api/skills/{skill_id}/start
```

### 停止应用
```
POST /api/skills/{skill_id}/stop
```

### 获取状态
```
GET /api/skills/{skill_id}/status
```

### 查看日志
```
GET /api/skills/{skill_id}/logs?lines=100
```

### 打开应用
```
POST /api/skills/{skill_id}/open
```

### 获取所有运行中的应用
```
GET /api/system/processes
```

## 📁 进程管理

### 进程信息存储

```
/tmp/skill-dashboard-processes.json
```

包含：
- PID (进程 ID)
- 脚本名称
- 端口号
- 启动时间
- 日志文件路径

### 日志文件

```
/tmp/skill-{skill_id}.log
```

## ⚠️ 注意事项

1. **权限**: 确保 Skill 目录有执行权限
2. **依赖**: 确保已安装所需 Python 包
3. **端口**: 确保端口未被占用
4. **日志**: 启动失败时查看日志文件
5. **停止**: 停止应用会终止整个进程组

## 🐛 故障排查

### 启动失败

**检查日志**：
```bash
cat /tmp/skill-{skill_id}.log
```

**常见错误**：
- 缺少依赖包 → `pip install -r requirements.txt`
- 端口被占用 → 修改端口配置
- 权限不足 → `chmod +x script.py`

### 无法打开浏览器

**检查**：
1. 应用是否运行中
2. 端口是否正确
3. 防火墙是否阻止

### 日志为空

**可能原因**：
- 应用未输出日志
- 日志文件路径错误
- 权限问题
