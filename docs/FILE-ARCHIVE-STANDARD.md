# 文件归档与发送规范

> 版本：v1.0 | 创建：2026-04-08 | 执行：自动

---

## 📁 归档目录结构

```
workspace/
├── reports/          # 报告类（日报/周报/月报/专项报告）
├── design/           # 设计类（DESIGN.md/风格指南/视觉规范）
├── docs/             # 文档类（API 文档/使用说明/教程）
├── memory/           # 记忆类（核心记忆/原始日志）
├── skills/           # 技能类（新 Skill/技能更新）
├── constitution/     # 宪法类（宪法修订/指令更新）
├── data/             # 数据类（CSV/JSON/数据集）
├── tmp/              # 临时类（一次性文件/草稿）
└── media/            # 媒体类（图片/视频/音频）
```

---

## 📤 微信发送规则

### 自动发送
| 文件类型 | 发送条件 | 发送方式 |
|----------|----------|----------|
| **报告类** | 生成完成 | 自动发送文件 |
| **设计类** | 生成完成 | 自动发送文件 |
| **文档类** | 用户要求 | 自动发送文件 |
| **记忆类** | 用户要求 | 自动发送文件 |
| **技能类** | 创建完成 | 自动发送文件 |
| **宪法类** | 修订完成 | 自动发送文件 |

### 不自动发送
| 文件类型 | 原因 |
|----------|------|
| **临时类** | 草稿/一次性，无需归档 |
| **系统类** | AGENTS.md/SOUL.md 等，用户已知 |
| **大文件** | >10MB，需用户确认 |

## 📝 文件命名规范

### 报告类
```
reports/YYYY-MM-DD-daily.md       # 日报
reports/YYYY-MM-weekly.md         # 周报
reports/YYYY-MM-monthly.md        # 月报
reports/YYYY-MM-DD-[主题].md      # 专项报告
```

### 设计类
```
design/[项目名]-design.md         # 设计文档
design/[项目名]-style-guide.md    # 风格指南
```

### 文档类
```
docs/[模块名]-api.md              # API 文档
docs/[功能名]-guide.md            # 使用指南
docs/[主题]-tutorial.md           # 教程
```

### 记忆类
```
memory/YYYY-MM-DD.md              # 原始日志
memory/core.md                    # 核心记忆
memory/residual.md                # 残差记忆
```

### 技能类
```
skills/[技能名]/SKILL.md          # 技能定义
skills/[技能名]/README.md         # 技能说明
```

### 宪法类
```
constitution/directives/[名称].md # 指令
constitution/extensions/[名称].md # 扩展
constitution/axiom/[名称].md      # 公理
```

## 🎨 文件美学规范

### Markdown 结构
```markdown
# 标题

> 一句话摘要/引言

## 章节

内容...

### 小节

- 列表项
- 列表项

## 总结

核心结论...
```

### 表格规范
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 内容 | 内容 | 内容 |

- 表头清晰
- 对齐一致
- 不拥挤

### 代码块
```python
# 有注释的代码
def function():
    pass
```

- 语言标识
- 必要注释
- 空行分组

## 📤 微信发送流程

```
1. 文件生成完成
2. 验证文件存在
3. 通过微信通道发送文件
4. 回复确认消息（含文件路径）
```

### 确认消息格式
```
✅ [文件类型] 已生成并发送

📁 路径：workspace/[目录]/[文件名].md
📊 大小：[文件大小]
🎨 风格：[应用的设计风格]
```

## 🔄 版本控制

- 所有文件纳入 Git 管理
- 每次修改后自动 commit
- Commit 信息格式：`[类型] 描述 - 日期`

示例：
```
git add .
git commit -m "[design] 创建太一设计系统 - 2026-04-08"
git push
```
