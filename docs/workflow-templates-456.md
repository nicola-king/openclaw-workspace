## 工作流 4：周末家庭协调

**负责 Bot**：守藏吏 + 山木

**触发条件**：每周六 08:00

**流程**：

```
┌─────────────────────────────────────────┐
│ 1. 读取家庭日历（24-48h 内事件）         │
├─────────────────────────────────────────┤
│ 2. 识别冲突（时间/地点/人员）           │
├─────────────────────────────────────────┤
│ 3. 生成协调方案（备选时间/替代方案）     │
├─────────────────────────────────────────┤
│ 4. 发送微信消息给家庭成员               │
├─────────────────────────────────────────┤
│ 5. 收集反馈并确认最终方案               │
├─────────────────────────────────────────┤
│ 6. 更新日历并设置提醒                   │
└─────────────────────────────────────────┘
```

**命令示例**：
```bash
# 手动触发
openclaw workflow family-coord --weekend 2026-04-04

# 定时任务（Cron）
0 8 * * 6 /home/nicola/.openclaw/workspace/scripts/family-coord.sh
```

**输出模板**：
```
【周末家庭协调 · 2026-04-05】

📅 已确认事件：
- 09:00 孩子钢琴课（地点：XX 琴行）
- 14:00 家庭聚餐（地点：XX 餐厅）

⚠️ 时间冲突：
- 15:30 孩子足球课 vs 家庭聚餐（延长）
  建议：爸爸带孩子去足球课，妈妈继续聚餐

✅ 待确认：
- 周日 10:00 郊游（需确认天气）

📲 已发送消息给：爸爸、妈妈
```

## 工作流 5：销售线索筛选

**负责 Bot**：罔两 + 知几

**触发条件**：GitHub Issue 创建 + Webhook

**流程**：

```
┌─────────────────────────────────────────┐
│ 1. 接收 GitHub Issue Webhook            │
├─────────────────────────────────────────┤
│ 2. 分析 Issue 内容（需求/预算/紧急度）   │
├─────────────────────────────────────────┤
│ 3. 评分（0-100，基于转化概率）          │
├─────────────────────────────────────────┤
│ 4. 分级（P0/P1/P2）                     │
├─────────────────────────────────────────┤
│ 5. 分配 Bot 处理（太一/山木/素问）       │
├─────────────────────────────────────────┤
│ 6. 发送跟进消息（微信/邮件）            │
└─────────────────────────────────────────┘
```

**评分标准**：
| 因素 | 权重 | 评分 |
|------|------|------|
| 需求明确度 | 30% | 清晰=30，模糊=10 |
| 预算匹配 | 30% | 匹配=30，未知=15 |
| 紧急度 | 20% | 紧急=20，一般=10 |
| 决策权 | 20% | 有决策权=20，无=5 |

**分级标准**：
- **P0（≥80 分）**：太一亲自处理，2h 内响应
- **P1（60-79 分）**：山木处理，24h 内响应
- **P2（<60 分）**：自动回复 + 资料发送

**命令示例**：
```bash
# 手动评分
openclaw workflow lead-score --issue 123

# Webhook 自动触发
# GitHub → OpenClaw Webhook → 自动处理
```

**输出模板**：
```
【销售线索评分 · Issue #123】

📊 评分结果：
- 需求明确度：30/30（清晰描述定制需求）
- 预算匹配：25/30（预算¥5000，匹配 P0）
- 紧急度：20/20（本周内上线）
- 决策权：20/20（创始人直接联系）
- **总分：95/100**

🎯 分级：P0（太一亲自处理）

📲 已发送消息：
- 微信：已发送报价单
- 邮件：已发送案例集

⏰ 下一步：
- 2026-04-02 14:00 电话会议
- 2026-04-03 12:00 前发送合同
```

## 工作流 6：会议准备

**负责 Bot**：山木 + 知几

**触发条件**：日历事件前 2h

**流程**：

```
┌─────────────────────────────────────────┐
│ 1. 读取日历事件（2h 后开始的会议）       │
├─────────────────────────────────────────┤
│ 2. 收集参会人信息（GitHub/领英/微信）   │
├─────────────────────────────────────────┤
│ 3. 搜索背景资料（公司/项目/最近动态）   │
├─────────────────────────────────────────┤
│ 4. 准备会议议程（基于历史沟通）         │
├─────────────────────────────────────────┤
│ 5. 生成会议简报（1 页 PDF）              │
├─────────────────────────────────────────┤
│ 6. 发送微信/邮件给参会人                │
└─────────────────────────────────────────┘
```

**会议简报内容**：
1. 会议基本信息（时间/地点/参会人）
2. 参会人背景（公司/职位/项目）
3. 历史沟通记录（上次会议要点）
4. 本次会议目标（议程/预期结果）
5. 参考资料（链接/文档）

**命令示例**：
```bash
# 手动触发
openclaw workflow meeting-prep --event 2026-04-02-1400

# 定时任务（Cron 每小时检查）
0 * * * * /home/nicola/.openclaw/workspace/scripts/meeting-prep.sh
```

**输出模板**：
```
【会议准备 · 2026-04-02 14:00】

📅 会议信息：
- 主题：OpenClaw 定制开发讨论
- 时间：2026-04-02 14:00-15:00
- 地点：腾讯会议（链接：xxx）
- 参会人：张三（XX 公司 CTO）、李四（技术负责人）

👥 参会人背景：
- 张三：XX 公司 CTO，10 年技术经验，GitHub: @zhangsan
- 李四：技术负责人，前阿里 P8，专注 AI 基础设施

📝 历史沟通：
- 2026-03-25：首次接触，了解 OpenClaw 功能
- 2026-03-28：发送报价单（¥20000 企业部署）
- 2026-04-01：确认意向，安排技术会议

🎯 本次会议目标：
1. 技术需求确认（部署环境/定制功能）
2. 时间线确认（4 周上线）
3. 合同条款讨论

📚 参考资料：
- OpenClaw 文档：https://docs.openclaw.ai
- 报价单：[附件]
- 案例集：[附件]

✅ 已发送：
- 会议链接（微信）
- 会议简报（邮件）
```
