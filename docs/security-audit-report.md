## 1. 安全架构

### 1.1 权限模型

| 层级 | 权限 | 说明 |
|------|------|------|
| **用户** | 完全控制 | 可修改所有配置/代码 |
| **太一** | 统筹调度 | 可分配任务给 Bot |
| **Bot** | 职责域内 | 仅限专业领域操作 |
| **Gateway** | 消息路由 | 无业务逻辑权限 |

### 1.2 安全红线

**宪法级约束**（不可绕过）：

1. **数据不泄露**
   - 不主动发送用户数据到外部
   - 不存储明文密码/API Key
   - 不记录敏感对话

2. **命令安全分级**
   - LOW（只读）：status/list/search
   - MEDIUM（可逆）：add/update/send
   - HIGH（需审批）：delete/restart/exec

3. **可恢复优先**
   - trash > rm
   - 备份 > 直接修改
   - 日志 > 静默执行

|------|------|------|------|---------|

| R004 | 记忆文件未加密 | 🟢 低 | ✅ 可接受 | 本地存储 |
| R005 | Bot 越权调用 | 🟢 低 | ✅ 已防护 | 职责域隔离 |

### 2.2 风险详解

#### R001：API Key 明文存储

**现状**：
- 微信 Token 存储在环境变量
- 飞书 AppSecret 存储在配置文件
- GitHub Token 存储在.git/config

**风险**：
- 文件泄露导致凭证暴露
- 备份时可能同步到云端

**建议**：
1. 使用加密密钥管理工具（如 HashiCorp Vault）
2. 或至少使用文件权限保护（chmod 600）
3. 定期轮换凭证

**优先级**：P1（2 周内）

---

#### R002：外部 API 调用无审计

**现状**：
- web_search 调用 DuckDuckGo
- image_generate 调用外部模型
- 无详细调用日志

**风险**：
- 无法追溯敏感查询
- 无法审计 API 费用
- 无法检测异常调用

**建议**：
1. 创建 `logs/api-calls.log`
2. 记录：时间/API/参数/响应大小
3. 每日汇总报告

**优先级**：P1（2 周内）

---

#### R003：Webhook 无签名验证

**现状**：
- GitHub Webhook 无签名验证
- 任何人均可触发

**风险**：
- 恶意触发工作流
- DDoS 攻击

**缓解**：
- 当前仅内网部署，风险低
- 建议：添加 HMAC 签名验证

**优先级**：P2（1 月内）

1. **立即隔离**：停止受影响服务
2. **记录证据**：保存日志/截图
