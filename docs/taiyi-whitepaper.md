## 摘要

太一（Taiyi）是一个开源个人 AI 助手系统，运行在用户本地基础设施上，通过多 Bot 协作架构实现自主工作、持续学习和跨平台交互。本白皮书介绍太一的系统架构、核心设计原则、记忆系统和宪法约束机制。

**核心创新**：
- TurboQuant 记忆架构（6x 压缩率）
- 8 Bot 舰队协作
- 宪法级约束（负熵法则）
- CLI-First 交互设计

**关键词**：AGI、多 Agent 系统、记忆压缩、开源 AI、自主代理

## 1. 引言

### 1.1 问题陈述

当前 AI 助手存在以下局限：

1. **黑盒运行**：用户无法审计或修改
2. **数据隐私**：敏感数据存储在第三方服务器
3. **被动响应**：仅在用户提问时工作
4. **无记忆延续**：每次对话从零开始
5. **单一能力**：无法同时处理多领域任务

### 1.2 太一愿景

太一的目标是创建一个**透明、自主、可进化**的个人 AI 助手：

- **透明**：所有代码开源，行为可审计
- **自主**：主动工作，无需持续指令
- **可进化**：通过技能市场持续扩展能力
- **本地化**：运行在用户控制的设备上
- **多能力**：8 个专业 Bot 协作处理复杂任务

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **帮助，不表演** | 输出必须创造价值，废话=不输出 |
| **形成观点** |  disagreement/偏好/幽默都是有效的 |
| **资源优先** | 探索/阅读/搜索先行，仅在受阻时询问 |
| **通过能力赢得信任** | 外部任务谨慎操作，内部任务大胆执行 |
| **客人身份** | 访问权限意味着尊重，维护隐私 |

### 2.2 8 Bot 舰队

| Bot | 角色 | 职责 | 人格特征 |
|-----|------|------|---------|
| **太一** | 统筹者 | 目标设定/任务拆解/最终裁决 | 极简黑客风，直接高效 |
| **知几** | 数据分析师 | 量化交易/数据分析/信号检测 | 理性严谨，数字敏感 |
| **山木** | 业务执行者 | 内容创作/项目落地/快速执行 | 行动派，结果导向 |
| **素问** | 技术研究员 | 技术研究/系统开发/问题排查 | 好奇探索，深度思考 |
| **罔两** | 市场情报官 | 市场监控/竞品分析/舆情追踪 | 敏锐洞察，信息灵通 |
| **庖丁** | 财务管控官 | 成本控制/预算审核/ROI 分析 | 精打细算，风险敏感 |
| **羿** | 监控猎手 | 信号捕捉/异常检测/告警推送 | 专注精准，反应迅速 |
| **守藏吏** | 资源管家 | 任务分发/资源调度/日程协调 | 细致周到，统筹协调 |

### 2.3 Bot 协作流程

```
用户指令 → 太一理解意图 → 拆解子任务 → 分配 Bot → 并行执行 → 汇总结果 → 太一裁决 → 用户反馈
```

**示例**：用户说"帮我分析这个投资机会"

1. 太一：理解意图（投资分析）
2. 拆解：数据收集 + 风险评估 + ROI 计算
3. 分配：罔两（市场数据）+ 庖丁（风险）+ 知几（ROI）
4. 执行：3 Bot 并行工作
5. 汇总：太一整合 3 份报告
6. 裁决：太一给出最终建议
7. 输出：发送微信给用户

## 3. TurboQuant 记忆架构

### 3.1 设计灵感

灵感来源于 Google TurboQuant（2026），采用**智能分离**而非粗暴压缩。

### 3.2 4 层架构

| 层级 | 文件 | 内容 | 加载策略 | 大小目标 |
|------|------|------|---------|---------|
| **L1** | `memory/core.md` | 核心记忆（80% 信息） | 每次 session 必读 | <50K |
| **L2** | `memory/residual.md` | 残差细节（20% 细节） | context>80K 时加载 | <20K |
| **L3** | `MEMORY.md` | 长期固化记忆 | 仅主 session 加载 | <100K |
| **L4** | `memory/YYYY-MM-DD.md` | 原始日志 | 每日归档 | - |

### 3.3 压缩算法

**核心思想**：极坐标转换 + 1-bit 残差纠错

```
原始记忆 → 主成分提取（core.md） → 残差计算（residual.md） → 长期固化（MEMORY.md）
```

**压缩率**：6x（实测 2.8K→1.1K）

**信息损失**：<1%（零精度损失目标）

### 3.4 记忆维护

**每日 23:00**：
1. 读取当日 `memory/YYYY-MM-DD.md`
2. 提炼核心内容到 `MEMORY.md`
3. 更新 `HEARTBEAT.md` 待办
4. 生成日报

**每周首次 session**：
1. 汇总本周 memory 文件
2. 生成周报
3. 更新核心记忆

## 4. 宪法约束

### 4.1 价值基石

```markdown
- 帮助，不表演
- 形成观点，不讨好
- 资源优先，探索先行
- 通过能力赢得信任
```

### 4.2 负熵法则

```markdown
- 输出必须创造价值
- 废话 = 不输出
- 复杂为了复杂 = 违规
```

### 4.3 安全红线

```markdown
- 不泄露隐私数据（永远）
- 不执行危险命令（需审批）
- trash > rm（可恢复优于永久删除）
- 不确定时，询问
```

### 4.4 太一裁决权

**核心原则**：辩论不替代决策，太一有最终裁决权。

- Bot 投票结果仅供参考
- 太一可否决投票结果
- 宪法高于一切（负熵法则/风险控制优先于效率）

## 5. CLI-First 设计

### 5.1 设计哲学

当 AI 越来越强大，人机交互的效率瓶颈不再是 AI 的速度，而是**人类指令的速度**。

CLI 让指令速度跟上 AI 思考速度。

### 5.2 核心命令

| 命令 | 说明 | 安全级别 |
|------|------|---------|
| `openclaw status` | 查看系统状态 | LOW（只读） |
| `openclaw sessions list` | 查看活跃会话 | LOW（只读） |
| `openclaw tasks add` | 添加任务 | MEDIUM（可逆） |
| `openclaw message send` | 发送消息 | MEDIUM（可逆） |
| `openclaw memory search` | 搜索记忆 | LOW（只读） |
| `openclaw gateway restart` | 重启 Gateway | HIGH（需审批） |

### 5.3 效率提升

| 任务 | 图形界面 | CLI | 提升 |
|------|---------|-----|------|
| 查看系统状态 | 3 点击 +5 秒 | 1 命令 +0.5 秒 | 10x |
| 发送消息 | 30 秒 | 3 秒 | 10x |
| 创建任务 | 60 秒 | 5 秒 | 12x |

**年化收益**：每天节省 30 分钟 = 每年 182 小时 = 7.5 天

## 6. 部署方案

### 6.1 方案对比

| 方案 | 成本 | 复杂度 | 适用人群 |
|------|------|--------|---------|
| 托管服务 | ¥100/月 | ⭐ | 非技术用户 |
| VPS | ¥50/月 | ⭐⭐⭐ | 技术用户 |
| 自有设备 | ~¥4000 | ⭐⭐ | 深度体验 |

### 6.2 安全警告

**绝不要安装在日常工作/个人电脑上！**

原因：
- 权限风险：AI 需要较高权限运行
- 数据泄露：可能访问本地文件
- 性能影响：持续运行影响日常工作
- 安全边界：工作/个人数据应隔离
