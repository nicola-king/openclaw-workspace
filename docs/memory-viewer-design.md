# 太一记忆可视化面板设计方案

**设计时间**: 2026-03-30 20:50
**设计人**: 太一 (素问协助)
**参考**: MemOS Memory Viewer

---

## 🎯 核心需求

1. **记忆浏览** - 查看 core.md + residual.md 内容
2. **搜索过滤** - 关键词搜索、日期过滤、标签过滤
3. **手动编辑** - 修正错误记忆
4. **删除功能** - 清理过期记忆
5. **统计面板** - Token 使用、记忆数量、压缩率

---

## 🏗️ 技术选型

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| **Feishu 多维表格** | 快速搭建、支持 API、可视化好 | 需飞书账号 | ✅ 采用 |
| **Web 面板** | 自定义高、独立部署 | 开发成本高 | ❌ 暂缓 |
| **CLI 工具** | 简单、无需额外依赖 | 可视化弱 | ❌ 不采用 |

---

## 📋 表格结构设计

### 主表：记忆条目 (memory_entries)

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| **id** | 自动编号 | 唯一 ID | 1001 |
| **记忆内容** | 多行文本 | 记忆正文 | "MemOS 学习完成..." |
| **来源文件** | 单选 | core/residual/daily | core.md |
| **日期** | 日期 | 记忆创建日期 | 2026-03-30 |
| **时间** | 时间 | 记忆创建时间 | 20:47 |
| **类型标签** | 多选 | [决策][任务][洞察][能力涌现] | [决策][学习] |
| **关键词** | 文本 | 搜索关键词 | MemOS,Token,可视化 |
| **重要度** | 单选 | P0/P1/P2 | P0 |
| **状态** | 单选 | 有效/待验证/已过期 | 有效 |
| **Token 估算** | 数字 | 记忆占用 Token 数 | 250 |
| **创建会话** | 文本 | 来源 Session ID | agent:taiyi:wechat |
| **最后修改** | 最后修改时间 | 自动记录 | 2026-03-30 20:50 |

### 统计表：记忆概览 (memory_dashboard)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **总记忆数** | 数字 | 所有记忆条目数量 |
| **core.md 数量** | 数字 | 核心记忆数量 |
| **residual.md 数量** | 数字 | 残差记忆数量 |
| **今日新增** | 数字 | 今天创建的记忆数 |
| **本周新增** | 数字 | 本周创建的记忆数 |
| **总 Token 占用** | 数字 | 所有记忆 Token 总和 |
| **平均压缩率** | 百分比 | TurboQuant 压缩效果 |
| **P0 记忆数** | 数字 | 高优先级记忆数量 |
