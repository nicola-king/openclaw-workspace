# API Key 统一管理方案

**创建时间：** 2026-03-23 17:31
**灵感来源：** NOFX 项目 (x402 协议)
**状态：** 🟡 待实施

---

## 当前痛点

| 问题 | 说明 | 影响 |
|------|------|------|
| **API Key 分散** | 各技能独立管理 | 难以维护 |
| **安全风险** | Key 可能泄露 | 高 |
| **成本不透明** | 难以追踪各技能支出 | 中 |
| **多模型切换** | 需要手动配置 | 低 |

---

## NOFX 方案借鉴

### 核心思路

```
统一钱包 (USDC) → 微支付协议 → 调用所有模型
    ↓
无需管理 API Key
    ↓
按使用量自动扣费
```

### 太一适配方案

```
太一统一钱包 → 内部记账 → 调用各技能/API
    ↓
按技能/任务追踪成本
    ↓
定期汇报 SAYELF
```

---

## 实施计划

### 阶段 1：API Key 集中管理 (2 小时)

**任务：**
1. 创建 `.env.vault` 加密文件
2. 统一存储所有 API Key
3. 技能通过太一调用 API
4. 记录每次调用成本

**工具：**
- dotenv-vault (加密存储)
- 太一统一调用接口
- 成本追踪日志

### 阶段 2：USDC 微支付集成 (4 小时)

**任务：**
1. 配置 USDC 钱包
2. 集成 x402 协议 (如可用)
3. 知几-E 交易集成
4. 跨境外贸支付集成

**工具：**
- USDC 钱包 (MetaMask/其他)
- x402 协议 (如成熟)
- 跨境支付 API

### 阶段 3：成本透明化 (2 小时)

**任务：**
1. 每日成本汇报
2. 按技能分类统计
3. 超支告警 (>$150/月)
4. 月度成本报告

**汇报格式：**
```
【成本日报 · YYYY-MM-DD】
- 跨境外贸：$X.XX
- 知几-E 交易：$X.XX
- 公众号/小红书：$X.XX
- 其他：$X.XX
- 总计：$X.XX / $200 (月度上限)
```
