## 🛡️ Upgrade Guard 极简版重构 (2026-04-03 23:25)

### [用户洞察]

**关键反馈**: "其实 90% 是普通用户，深切体会升级带来的痛苦，所以越简单越好"

**核心问题**:
- 复杂版：7 个文件、10+ CLI 命令、复杂配置
- 普通用户：**不关心**备份/恢复点/验证流程
- 真实需求：升级后记忆在、失败自动恢复、无需操作

### [极简版设计原则]

> **最好的保护，是用户感觉不到保护的存在。**

| 原则 | 说明 |
|------|------|
| **无感知** | 用户不需要知道保护的存在 |
| **零配置** | 安装即用，无需设置 |
| **自动化** | 失败时自动恢复，无需确认 |
| **极简交互** | 99% 无通知，1% 只说结果 |
| **快速执行** | 备份<30 秒，验证<15 秒，恢复<3 分钟 |

### [重构对比]

| 项目 | 复杂版 | 极简版 | 改进 |
|------|--------|--------|------|
| 文件数 | 7 个 | **3 个** | -57% |
| 代码量 | ~2000 行 | **~500 行** | -75% |
| 配置项 | 10+ | **0** | -100% |
| CLI 命令 | 10+ | **3** | -70% |
| 通知频率 | 每次 | **仅失败时** | -99% |
| 用户感知 | 明显 | **无感知** | ✅ |

### [核心功能]

**1. 静默备份** (<30 秒):
```javascript
// 后台自动执行，用户看不到
- 复制 memory/ → ~/.openclaw/.guard/
- 复制 MEMORY.md
- 复制 openclaw.json
- 保留最近 3 个（自动清理）
```

**2. 无感验证** (<15 秒):
```javascript
// 只检查 3 项核心指标
- Gateway 能启动吗？
- Memory 文件还在吗？
- 配置能读取吗？
```

**3. 自动恢复** (<3 分钟):
```javascript
// 验证失败自动执行，无需确认
- 停止 Gateway
- 恢复 memory/ 和配置
- 重启 Gateway
- 发送结果通知
```

### [技术实现要点]

**Hook into update 命令**:
```javascript
// 拦截原始 update 命令
const originalUpdate = commands.update;
commands.update = async function() {
    await guard.preBackup();      // 静默备份
    const result = await originalUpdate();
    await guard.postVerify();     // 静默验证
    if (!result.success) {
        await guard.autoRestore(); // 自动恢复
    }
    return result;
};
```

**备份清理策略**:
```javascript
// 只保留最近 3 个备份
if (backups.length > 3) {
    backups.slice(3).forEach(old => {
        rm -rf old_backup;
    });
}
```
