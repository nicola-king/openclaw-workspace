## 📊 已创建模块

### 1. 预测性维护模块

**文件**: `predictive_maintenance.py`

**功能**:
```python
✅ 基于趋势预测故障
✅ 提前告警 (60 分钟前)
✅ 预防性维护建议
✅ 趋势分析:
   - rapid_increase (快速上升)
   - slow_increase (缓慢上升)
   - stable (稳定)
   - slow_decrease (缓慢下降)
   - rapid_decrease (快速下降)
```

**预测逻辑**:
```python
1. 收集历史数据 (至少 5 个点)
2. 线性回归计算斜率
3. 预测未来值
4. 判断是否超过阈值
5. 生成维护计划
```

**示例**:
```
CPU 历史：[30, 32, 35, 38, 42, 45, 48, 52, 55, 58]
趋势：rapid_increase (斜率：3.07)
预测：60 分钟后达到警告值 (80%)
建议：提前处理 CPU 使用率上升问题
```

### 2. 自动阈值调整模块

**文件**: `auto_threshold_adjustment.py`

**功能**:
```python
✅ 基于历史数据自动调整
✅ 减少误报
✅ 优化告警质量
✅ P95/P99 统计计算
✅ 调整幅度限制 (±20%)
```

**调整逻辑**:
```python
1. 收集历史数据 (至少 20 个点)
2. 计算 P95 和 P99
3. 计算最优阈值 = (P95 + P99) / 2
4. 限制调整幅度 (±20%)
5. 生成调整建议
```

**示例**:
```
CPU 历史：[30, 32, ..., 80, 82] (20 个点)
当前阈值：80
P95: 75, P99: 80
最优阈值：77.5
调整建议：80 → 77.5 (-3.1%)
```
