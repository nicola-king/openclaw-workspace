# 太一操作工作站/笔记本授权协议

> 明确授权机制，确保 SAYELF 有最终控制权

---

## 🎯 核心原则

```
1. 太一可以操作工作站和笔记本
2. 但需要 SAYELF 授权
3. 授权级别决定授权方式
4. 所有操作记录在案
```

---

## 📋 授权级别

| 级别 | 名称 | 授权方式 | 额度 | 响应 |
|------|------|---------|------|------|
| **P0** | 紧急任务 | 事后报备 | 无限制 | 立即执行 |
| **P1** | 高优先级 | Telegram 确认 | <¥100/次 | <10 分钟 |
| **P2** | 普通任务 | 周批量授权 | <¥500/周 | <1 小时 |
| **P3** | 低优先级 | 等待询问 | 无预算 | 等待 |

---

## 🔐 授权流程

### P0 紧急任务

```
太一检测到紧急情况
    ↓
立即执行
    ↓
Telegram 通知 SAYELF
    ↓
写入授权日志
    ↓
等待 SAYELF 确认收到
```

**适用场景**:
- 系统故障恢复
- 安全告警响应
- Gateway 重启

---

### P1 高优先级

```
太一创建任务

    ↓
SAYELF 回复确认
    ↓

    ↓
执行后报告结果
```

**Telegram 消息**:

## 🚨 安全机制

### 紧急停止

SAYELF 发送以下任一消息:
- `/stop`
- `停止`
- `紧急停止`

**效果**:
```
1. 立即停止所有 P1/P2/P3 任务
2. P0 任务继续 (安全相关)
3. 等待进一步指令
```

### 敏感操作二次确认

以下操作需要二次确认:
- 删除文件 (>100MB)
- 安装软件
- 修改系统配置
- 访问敏感数据
- 对外发送数据
