## 🔧 自动接管协议

### 太一 ↔ 守藏吏

```yaml
监控：
  - 心跳检查：每 5 分钟
  - 故障判定：连续 3 次无响应
  
接管流程:
  1. 守藏吏检测到太一无响应
  2. 发送告警到 Telegram
  3. 接管定时任务调度
  4. 维持基本服务 (备份/监控)
  5. 等待太一恢复
  
恢复流程:
  1. 太一恢复后自检
  2. 从守藏吏同步状态
  3. 重新接管指挥权
  4. 记录故障日志
```

---

### 知几 ↔ 元龟

```yaml
监控:
  - 交易信号检查：每 30 分钟
  - 故障判定：错过 2 次交易窗口
  
接管流程:
  1. 元龟检测到知几无信号
  2. 切换到保守模式 (Quarter-Kelly→Half-Kelly)
  3. 暂停新开仓，仅平仓
  4. 发送告警
  
恢复流程:
  1. 知几恢复后验证信号
  2. 从元龟同步持仓状态
  3. 恢复正常交易
```

---

### 山木 ↔ 蝴蝶

```yaml
监控:
  - 内容发布检查：每日 09:00
  - 故障判定：错过发布窗口
  
接管流程:
  1. 蝴蝶检测到山木未发布
  2. 使用预生成内容库
  3. 发布简化版内容
  4. 发送告警
```

---

### 素问 ↔ 梓竹

```yaml
监控:
  - 代码提交检查：每周
  - 故障判定：技能开发停滞>3 天
  
接管流程:
  1. 梓竹检测到素问停滞
  2. 接管待开发任务
  3. 优先维护现有技能
  4. 暂停新功能开发
```

| 等级 | 故障容忍 | 恢复时间 | 目标 |
