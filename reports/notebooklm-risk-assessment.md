## ⚠️  核心风险分析

### 1. 认证安全风险

**问题**: 使用 Google Cookie (`__Secure-1PSID`) 进行认证

**风险点**:
- ❌ Cookie 泄露 = 账号被盗
- ❌ Cookie 有效期长 (数周至数月)
- ❌ 可能被用于未授权访问

**缓解措施**:
```bash
# ✅ Cookie 存储在本地
~/.nlm/env (权限 600)

# ✅ 不上传到任何服务器

# ✅ 定期轮换 Cookie
nlm refresh

# ✅ 使用独立 Google 账号 (非主账号)
```

**建议**:
- 🔒 使用专用 Google 账号 (非个人主账号)
- 🔒 定期检查 Cookie 权限
- 🔒 不要在公共电脑使用

---

### 2. API 稳定性风险

**问题**: 使用 Google 内部 API (BatchExecute 协议)

**风险点**:
- ❌ 非官方 API，随时可能变更
- ❌ Google 可能封锁未授权访问
- ❌ 无官方支持/文档

**历史案例**:
- 2024: 多个第三方 NotebookLM 工具被封
- 2025: Google 加强 API 访问控制
- 2026: tmc/nlm 持续维护中

**缓解措施**:
- ✅ 关注 GitHub 更新 (star/watch 项目)
- ✅ 定期更新到最新版本
- ✅ 准备备用方案 (官方 Web 界面)

**建议**:
- 📌 不要用于生产环境关键流程
- 📌 重要数据定期备份到本地
- 📌 关注项目 Issues 和 Releases
