## 🎭 角色定义

### 分析师层（3 角色）

| 角色 | 职责 | 数据源 | 输出 |
|------|------|--------|------|
| **Fundamental Analyst** | 基本面分析 | 财报/宏观经济/行业数据 | 估值判断/长期趋势 |
| **Technical Analyst** | 技术面分析 | K 线/指标/形态 | 入场点/止损点 |
| **Sentiment Analyst** | 情绪面分析 | 新闻/社交媒体/舆情 | 市场情绪评分 |

### 交易员层（3 角色）

| 角色 | 职责 | 风险偏好 | 决策依据 |
|------|------|---------|---------|
| **Aggressive Trader** | 激进交易 | 高 | 技术信号 + 动量 |
| **Conservative Trader** | 保守交易 | 低 | 基本面 + 估值 |
| **Risk Manager** | 风控审核 | 极低 | 仓位控制/止损 |

## 🔄 协作流程（ReAct 框架）

### ReAct 循环

```
Reason → Act → Observe → Reason → Act → ...
```

**具体流程**：

1. **Reason（推理）**：Agent 分析当前状态，生成思考
2. **Act（行动）**：执行动作（查询数据/执行交易/发起辩论）
3. **Observe（观察）**：获取环境反馈
4. **Repeat（循环）**：直到达成决策或达到步数限制

### 多 Agent 协作

```
┌──────────────┐
│ 共享环境状态  │ ← 所有 Agent 可读/可写
└──────────────┘
       ↓
┌──────────────────────────────────────┐
│ Agent 1: 查询数据 → 分析 → 发布观点   │
├──────────────────────────────────────┤
│ Agent 2: 读取观点 → 质疑 → 补充论据   │
├──────────────────────────────────────┤
│ Agent 3: 综合观点 → 投票 → 执行决策   │
└──────────────────────────────────────┘
```

## 🗳️ 决策机制

### 辩论流程

1. **陈述阶段**：各 Agent 发布观点 + 论据 + 置信度
2. **质疑阶段**：Agent 之间交叉提问，指出缺陷
3. **修正阶段**：根据质疑调整观点
4. **投票阶段**：加权投票（权重=专业度×历史准确率）
5. **执行阶段**：按投票结果执行

### 置信度计算

```
最终置信度 = Σ(各 Agent 置信度 × 权重) / Σ权重

权重 = 基础权重 × 历史准确率修正
```

### 决策阈值

| 置信度 | 行动 |
|--------|------|
| ≥90% | 立即执行 |
| 70-89% | 建议执行 |
| 50-69% | 需确认 |
| <50% | 暂缓 |

## 🧠 记忆设计

### 共享环境状态

- **全局可见**：所有 Agent 可读取
- **按需加载**：避免信息过载
- **版本控制**：记录状态变更历史

### Agent 私有记忆

- **短期记忆**：当前任务上下文
- **长期记忆**：历史决策/经验教训
- **检索机制**：语义搜索 + 关键词匹配
