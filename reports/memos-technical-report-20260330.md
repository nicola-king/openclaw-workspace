## 💡 Token 节省 72% 原理

### OpenClaw 原生问题

1. **检索算法简陋**
   - 召回不稳定
   - 相关性弱
   - Agent 需要反复试错与重问
   - Token 随对话轮次快速累积

2. **上下文注入过量**
   - 每次执行固定读取近两天全量记忆
   - today + yesterday + 长期记忆
   - 大量无关内容注入

3. **无智能压缩**
   - 原始对话直接存储
   - 未提取核心信息
   - Token 浪费严重

### MemOS 解决方案

**核心思路**: 智能记忆检索 替代 全量历史加载

```
原生方式:
Context = 全量历史对话 (100% Token)

MemOS 方式:
Context = 检索相关记忆 (28% Token)

节省率 = (100% - 28%) / 100% = 72%
```

### 实现机制

| 机制 | 说明 | Token 节省贡献 |
|------|------|---------------|
| **任务总结** | 自动压缩对话为结构化记忆 | ~30% |
| **智能召回** | 按意图检索，不塞无关历史 | ~25% |
| **分级模型** | 简单任务用小模型 | ~10% |
| **记忆去重** | 避免重复内容注入 | ~7% |
| **合计** | - | **~72%** |

   - 不需要过多引导
