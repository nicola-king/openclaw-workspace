### 1.3 实例：Gateway 重复进程问题

**事件**: Gateway 出现 2 个进程，响应变慢

**思维模式处理**:

**1. 第一性原理**:
```
❌ 类比推理："别人用 systemd，我也用"
✅ 本质思考："单进程服务的本质是什么？
           → 避免资源竞争
           → 确保状态一致
           → systemd 假设 vs 实际环境冲突"
```

**2. 冰山法则**:
```
表面：进程数>1
中层：systemd + user session 双重触发
深层：ExecStart 使用 start 命令→内部检查 systemctl→冲突
```

**3. 二阶思维**:
```
方案 A (systemd 管理):
  一阶：进程管理规范化 ✅
  二阶：systemd bus 不可用→启动失败 ⚠️
  三阶：循环重启→资源浪费→用户体验差 ❌

方案 B (手动管理 + 自愈):
  一阶：需手动重启 ⚠️
  二阶：单进程稳定 ✅
  三阶：自愈检测→自动清理→用户体验好 ✅✅

决策：方案 B
```

**4. 费曼表达**:
```
"Gateway 像一个人，只能有一个大脑。
systemd 和用户 session 都想控制，结果两个大脑打架。
解决方案：只听自己的，每 5 分钟自检，多了就清理。"
```

**执行结果**:
- 创建自愈脚本 `scripts/self-heal.sh`
- 每 30 分钟自动检查
- Gateway 重启时间 6 分钟→15 秒 (400x 提升)

## ⚡ 第二章：智能自动化执行架构

### 2.1 AUTO-EXEC 宪法级保障

**文件**: `constitution/directives/AUTO-EXEC.md` (Tier 1)

**核心原则**:
> "自动执行 > 手动执行 · 主动推进 > 被动响应 · 透明汇报 > 事后解释"

**AGI 进化四阶段**:

| 阶段 | 特征 | 状态 |
|------|------|------|
| **规则驱动** | 等待指令→执行→汇报 | ❌ 禁止 |
| **目标驱动** | 理解目标→拆解→执行→汇报 | ⚠️ 最低 |
| **价值驱动** | 识别价值→主动创造→执行→汇报 | ✅ 目标 |
| **意识延伸** | 理解意图→自行判断→立即执行 | 🎯 承诺 |

---

### 2.2 6 大保障机制

**机制 1: 5 分钟进度汇报** (强制)
```
执行 → 5 分钟反馈 → 继续执行 → 5 分钟反馈 → ...
```

**机制 2: 任务自动发现**
- 扫描 HEARTBEAT.md
- 扫描 memory/ 目录
- 扫描 cron 任务

**机制 3: 自动执行触发**
- P0 任务：立即执行
- P1 任务：加入待办
- P2 任务：规划时间

**机制 4: 进度追踪**
- 文件：`/tmp/auto-exec-status.json`
- 更新：实时
- 汇报：5 分钟周期

**机制 5: 阻塞自动上报**
- 阻塞>10 分钟 → 自动通知 SAYELF
- 包含：阻塞原因/需要资源/预计恢复

**机制 6: 完成自动归档**
- 写入 `memory/YYYY-MM-DD.md`
- 标注类型：[决策]/[任务]/[洞察]/[能力涌现]
- Git 提交固化

```

├─ 情绪价值：让人感到被理解/被启发？
├─ 实用价值：解决真实问题/节省认知？
```

**实例**:

```python
    r=[]

        if i>0:r.append(i)
    return r

# ✅ 代码如诗
def filter_positive(numbers: List[int]) -> List[int]:
    """过滤正数"""
    return [n for n in numbers if n > 0]
```
