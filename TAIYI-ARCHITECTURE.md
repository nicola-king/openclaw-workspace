## 🧠 TurboQuant 记忆架构

```
┌─────────────────────────────────────────┐
│  MEMORY.md (长期记忆)                    │
│  - 核心洞察                              │
│  - 重要决策                              │
│  - 学习心得                              │
│  ← 仅主 Session 加载                     │
├─────────────────────────────────────────┤
│  memory/core.md (核心记忆)               │
│  - 80% 信息 (主成分)                     │
│  - P0 任务                               │
│  - 核心配置                              │
│  ← 每次 Session 必读                     │
├─────────────────────────────────────────┤
│  memory/residual.md (残差记忆)           │
│  - 20% 细节 (残差)                       │
│  - 技术细节                              │
│  - 历史数据                              │
│  ← context>80K 时加载                    │
├─────────────────────────────────────────┤
│  memory/YYYY-MM-DD.md (原始日志)         │
│  - 每日记录                              │
│  - 执行日志                              │
│  - 原始数据                              │
│  ← 恢复上下文用                          │
└─────────────────────────────────────────┘
```

**压缩原则**:
- 新记忆自动分类到 core 或 residual
- 每日回顾：residual → core → MEMORY.md 提炼
- core>50K 时触发压缩

```
│  监控层                                  │

└─────────────────────────────────────────┘
```

**核心原则**:
> 自动执行 > 手动执行 · 主动推进 > 被动响应
